# 路径级 SEC/SBC 数据铺设设计模板

**Status**: Draft（待 game design 输入）
**Date**: 2026-07-14
**Depends**: cv-005 校准报告（全局参数天花板 ~57%）

## 背景

adr-0010 三层防御漏斗已闭环（cv-006/007/008），但 SEC/SBC 全默认 1000（中性）。
cv-005 校准数据表明全局参数可从 73% 降到 ~57%，完全收敛需路径级非对称数据。

## SEC（闪避系数）设计指南

SEC = 招式闪避系数（permille 基准 1000=中性），`CombatSkillDef.Sec` 字段已就位。

| SEC 值 | 语义 | 范例路径 |
|---|---|---|
| 0 | 必中（无法闪避） | 神识锁定、诅咒 |
| 1-500 | 难闪（命中抬升） | 重击、暗器 |
| 500-1000 | 略难闪 | 快剑 |
| 1000 | 中性（默认） | 普通攻击 |
| 1000-2000 | 易闪 | 慢速法术 |
| 2000+ | 极难命中 | 范围轰炸（易被预判闪避） |

### 设计原则
- 核心招式（Ultimate）SEC 应偏低（100-500，难闪 = 威压感）
- 快速连击 SEC 可偏高（1000-2000，易闪但数量多 = 压制感）
- 控制类招式 SEC 应偏低（被控住就完了，不该被闪）
- **数据驱动**：每路 3-5 招设 SEC 值，验证后扩展至全招

## SBC（格挡系数）设计指南

SBC = 招式格挡系数（permille 基准 1000=中性），`CombatSkillDef.Sbc` 字段已就位。调制 Chip 穿透比例。

| SBC 值 | 语义 | 范例路径 |
|---|---|---|
| 0 | 不可格挡（全伤穿透） | 无形剑气、灵魂攻击 |
| 1-500 | 重锤破防（穿透抬升） | 钝器、重型武器 |
| 500-1000 | 略难格挡 | 宽刃剑 |
| 1000 | 中性（默认） | 普通攻击 |
| 1000-2000 | 易格挡（穿透降低） | 轻剑、柔劲 |
| 2000+ | 极易格挡 | 气劲远程（距离衰减+可被招架化解） |

### 设计原则
- 体修/横练类路径：高 SBC 招式（抗格挡）+ 低 SBC 招式（自身攻击难被格挡 = 重锤）
- 剑修/快攻类路径：中等 SBC（平衡）
- 法术类路径：SBC 差异大（火球高 SBC 可被格挡 vs 诅咒 SBC=0 不可格挡）
- **数据驱动**：每路 3-5 招设 SBC 值，验证后扩展

## 铺设策略

1. **Phase 1（MVP）**：选 3 条代表路（体修/剑修/法修），各设 3-5 招 SEC/SBC
2. **Phase 2（验证）**：重跑 cv-005 harness 验证违规下降
3. **Phase 3（全量）**：21 路全部铺设，目标 violations→0

## 技术实现

`CombatSkillDef` 已支持 SEC/SBC：
```csharp
new CombatSkillDef("skill_id", "技能名", tier, onUse, cost,
    DamageType.Normal,  // 伤害类型
    Sec: 500,           // 闪避系数（100=极难闪, 1000=中性, 2000=易闪）
    Sbc: 300            // 格挡系数（100=重锤破防, 1000=中性, 2000=易格挡）
)
```

当前 paths/*.cs 中所有 CombatSkillDef 使用定位构造未传 Sec/Sbc → 默认 1000（零改动惰性）。
铺设时只需在构造末尾加 `Sec: N, Sbc: M` 即可。

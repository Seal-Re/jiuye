## A.3 · 转职 / 觉醒 / 双修 + 平衡标定（新增）

### A3.1 转职 / 觉醒 / 双修（registry-research §4 转职五阶段 QBN 式）
- **三形态**（数据驱动 storylet，挂 A.2 奇遇框架）：①**转职**（换 PathId：触发条件[境界/机缘/灵根突变] → 新路，旧功法部分作废 + 新路从对应 realm 起；如剑修→剑仙、武夫→修真）；②**觉醒**（同路血脉/体质觉醒，解锁高阶功法类目或战力台阶，不换 PathId）；③**双修**（兼修第二路，受 slotCap/神识带宽限，战力取主路 + 副路加成，反噬风险）。
- **数据结构**：`TransitionDef{FromPathPred, ToPathId, Gate(境界/资源/机缘谓词), Carryover(保留/作废功法规则), Cost}`，挂 storylet（前置+效果），RngStream 5。开闭：加转职路线 = 加数据。
- **反噬 RiskModifier**（registry §5 可插拔，不入核心循环）：转职/双修/邪道速成的反噬作为 RiskModifier 装配，A.3 落地。
- 红线：转职改 PathId → CultivationState 迁移（resources/realm 映射），确定性；不破 INV-DET-Cult（快照纳转职态）。

### A3.2 平衡标定（INV-CROSS，最重，独立 calibration 子阶段）
- **INV-CROSS gate**（境界稿 §5，决策①）：每个 ≥2 条战斗路锚定的 UT，构造各路典型角色（四维生成期 Σ=80 中庸 + 该 UT realm + 标准功法配置），跑 N 次 1v1，断言两两胜率 ∈[40,60]%。
- **标定方法**：A.0 per-path RealmMultipliers/公式权重 build 期各自拟（critique 实测同 UT 24-70× 失真）。标定 = 以「同 UT 典型角色战力基准带」调每路 mul/权重收敛同台。**辅助路豁免必胜率**（设计战斗弱），但 UT 锚 = 真实战斗当量下限。
- **标定工具**（建议）：写「全 21 路 × 各 UT 典型角色战力矩阵」dump + INV-CROSS 批量对拍 harness，红黄绿可视，迭代调 Tuning（per-path 覆盖）。这是数值活，需多轮。
- **其他标定 INV**：INV-POP（人口带）、INV-FUNNEL（各路飞升率∈[1,K]）、INV-VARIETY（破单调）、致死率带。

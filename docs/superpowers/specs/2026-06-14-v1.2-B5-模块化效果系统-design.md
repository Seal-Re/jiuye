## 1. 问题与设计目标

**问题（规则审计实证）**：21 路设计层齐全（每路有专属战斗方式 + 5-8 招 + 签名机制），但**结构化层近乎空白**——机器只读 `Cost`/`AddResource`/`flag`；真机制（伤害公式/AOE/dot/克制/夺取/召唤/态切换）全埋 `Note`；`AddPenInteger.Amount` 全是占位威力数（自读 `EffectOp.cs:99` 证 `ComputeOnUseDelta` 只 `return op.Amount`）。

**目标**：
1. 战斗效果 = 可复用整数模块拼接，新功法 = 组合既有模块。
2. 每功法有机制（非纯数值），分稀有度（普通/稀有/唯一），越好越复杂（更多/更稀模块）。
3. 当前版本 21 路全可机器结算参战（分层全量），无 pure-flavor 空壳。
4. 全整数确定、不破 off 逐字节、有限算子集纪律（Special 逃逸口受控）。

## 2. 模块 schema（升级 `EffectOp`，向后兼容）

不新建平行类型——`EffectOp` 即"模块"原语（`ArtDef.Effects` / `CombatSkillDef.OnUse` 已是 `IReadOnlyList<EffectOp>`）。最小升级：

```csharp
public enum EffectTrigger { OnUse, OnDefend, Passive }   // 战斗期模块何时 fire；装配期算子忽略(走 ApplyPassive)
public enum EffectRarity  { Common, Rare, Unique }        // 稀有度

// 现: EffectOp(Kind, Key, Amount, Note) —— 几百处 4 参 positional 调用
public sealed record EffectOp(
    EffectOpKind Kind, string? Key, int Amount, string? Note,
    int Amount2 = 0,                                       // 次参: den(ratio=Amount/Amount2) 或 secondary 量
    EffectTrigger Trigger = EffectTrigger.OnUse,
    EffectRarity Rarity = EffectRarity.Common);
```

**向后兼容铁律**：新字段全 `= default` 且置于 `Note` 之后 → 现有 `new EffectOp(K, key, amt, note)` 4 参调用**原样编译**（21 路几百处不改）。新模块按需传扩展参。

**ratio 表达**：需 num/den 的机制用 `Amount`(num)/`Amount2`(den)，整数除（如燃血×4 = Amount=4,Amount2=1；anti_evil×3/2 = 3,2）。无 den 默认 Amount2=0 视为 1（避免除零，见 §8 钳制）。

## 3. 三档词汇表

> 词汇表 = 现 10 Kind + 11 新（标 `// L1` 新语义，spec §0 R5 不谎称零改核心）+ Special registry。新增战斗 Kind 在 `ComputeOnUseDelta`/R2 引擎有真分支执行。

### 3.1 普通档（Common，通用攻防，已有 4 算子，全路基础）
| Kind | 语义 |
|---|---|
| `AddPenInteger` | 固定破防量（基础攻） |
| `AddFlatDR` | 固定减伤 |
| `ScalarMul` | 整数 num/den 乘子 |
| `AddSituationalAdj` | 情境 % 修正 |

### 3.2 稀有档（Rare，新增参数化 Kind，跨多路复用）
| Kind(新) | 参数 | 语义 | trigger | 覆盖路 |
|---|---|---|---|---|
| `PenFromResource` | Key=资源, Amount/Amount2 | delta += res(Key)×num/den（合并"资源换爆发"：瞬时伤；持续抬战力仍走 AddTermWeightStep） | OnUse | 血体鬼魂魔毒蛊阵傀 |
| `SumOfSet` | Key=setKey, Amount/Amount2, **bandwidth cap** | delta += min(Σset, 神识带宽)×num/den（**Σ挂带宽上限**，网文补：防爆炸+还原低阶御不动剑阵） | OnUse | 器鬼阵蛊驭兽傀 |
| `AoePerTarget` | Amount=每敌量 | R2 单挑退化为 ×1（占位）；群战 ×敌数 → FULLSTRUCT | OnUse | 剑鬼法血符 |
| `CounterMul` | Key=tag, Amount/Amount2 | 防方带 tag(Key) 则 dmg×num/den（**倍乘档**；门控质变[无效化/反弹/收取]走 Special） | OnUse | 雷佛儒音法 |
| `DrainResource` | Key=资源, Amount | 防方 res(Key)−=量, 攻方 res(Key)+=量（夺运/夺养/落宝削资源/缚锋） | OnUse | 命魔血器毒蛊 |
| `Backlash` | Key=条件, Amount | 满足条件则攻方受 Amount 自伤（反震/承雷） | OnUse | 魂雷命因果 |
| `Dot` | Key=标记, Amount=每tick, Amount2=ticks | 防方挂 dot（本轮简版：回合标记，每回合掉 Amount，持续 Amount2 回合） | OnUse | 毒蛊音 |
| `Control` | Key=类型, Amount=回合 | 防方挂 control（本轮简版：下 Amount 回合该角色 selectMove 失效） | OnUse | 法鬼阵因果傀 |
| `ReflectDamage` | Amount/Amount2 | **网文补**：防方受击时攻方受 incoming×num/den（斗转星移/翻天印） | OnDefend | 体器佛(护宝罡) |
| `Evade` | Amount=闪避阈 | **确定性**：防方身法值 ≥ (攻方命中−阈) 则该击减免（D-EVADE，无骰，见 §5/§8） | OnDefend | 修轻功者 |

（资源换爆发"持续抬战力档"复用已有 `AddTermWeightStep`，不新增。）

### 3.3 唯一档（Unique，签名机制，Special 派发）
| Kind | 参数 | 语义 |
|---|---|---|
| `Special` | Key=handlerId | → `SpecialModuleRegistry[handlerId]` 派发 per-path 纯整数函数（§7） |

签名集（逐路迁，高价值先）：落宝夺器(器)/炸阵换爆发(阵)/夺舍续命(鬼魂魔毒蛊)/金身大成态(佛)/因果栈回溯(因果)/律场总门(音)/布阵门控(阵)/渡化退场(佛)…

### 3.4 "越好越复杂" 涌现
功法/招式 = 模块列表。复杂度 = 模块数 + 最高 rarity。普通功法 1-2 Common；顶级 = 多模块 + Rare/Unique。无需额外字段——观察模块列表即得。

## 4. trigger 模型 + R2 回合应用

复用 R2 稿 §4 骨架（HP=pe，3 回合同时结算，去先手偏置）。模块按 trigger 入不同相：

```
进战前:
  HP_A = HP_B = pe(校准战力, 耐久锚)
  双方 Passive 模块算入基线(底盘减伤/态乘子 num/den)
  各选本场招式(CombatSkills 中 tier≤realm & TryPayCost 付得起 & gate 满足; 简版取最高可用 tier)

每回合 r(0..2):
  # 1. 攻方 OnUse 模块逐个改这一击 raw 伤害(读 pre-HP, 列表顺序固定)
  dmgA = baseDmg(pe, Tier, atkStat)
  foreach m in 招式A.OnUse (Trigger==OnUse, 按列表序):
      apply_onuse(m, ctxA→B, ref dmgA)   # §3.2 各 Kind 真分支; Special→registry
  dmgB = 同上(招式B)

  # 2. 防方 OnDefend 模块改 incoming(护体/Evade/Reflect)
  applyDefense(B 的 OnDefend 模块, ref dmgA, ref reflectToA)
  applyDefense(A 的 OnDefend 模块, ref dmgB, ref reflectToB)

  # 3. 软情境相克(复用既有 SituationalResolver ±P0/4, 与硬 CounterMul 叠)
  dmgA = dmgA×(100+adj_A)/100; dmgB 同

  # 4. 同时结算(都算完才扣, 去先手偏置 — M6)
  HP_B -= max(0,dmgA)+reflect…; HP_A -= max(0,dmgB)+…

  # 5. 回合间: dot 掉血 / control 解限(简版回合标记)
  if HP_A<=0 || HP_B<=0: break

胜者 = HP 高者; 平 → Tiebreak(CharacterId 确定). 全程整数 clamp≥0.
越级(D-越级简版): UT 差≥2 → 高 UT 直接判胜(C2 碾压单调, 不进逐回合).
```

## 5. 功法门控防御（红线 §14.1 落地 + M5 反向门控）

防御能力 = **OnDefend 模块挂在修了的功法上**，stat 只调强弱不自带能力：

| 防御模式 | 谁有 | 模块 | 无功法时 |
|---|---|---|---|
| 硬抗(floor) | 人人 | baseDR = f(根骨) | —（体质天生） |
| **护体真气** | 修内功心法者 | OnDefend `AddFlatDR`/`PenFromResource`(减伤按内力) | **护体减伤 = 0** |
| **闪避** | 修轻功/身法者 | OnDefend `Evade`(Amount=闪避阈) | **闪避 = 0（没轻功不能闪）** |

- **Evade 确定性（D-EVADE）**：防方身法值（轻功功法提供）vs 攻方命中（招式+敏捷类 stat）→ 身法值 ≥ 命中−阈 则该击减免（全免或半免，§13 定）。**无骰**——sim 同局面同结果，免 determinism 风险、守 off gate、解 §4/§8 矛盾。
- **公共武技池**（R2 稿 §3.3，散修/任路可学）= 轻功(神行)/吐纳护体/基础招之家。"修轻功" = pick 公共池轻功功法 → 装 Evade 模块。散修靠此池有最低战斗能力。
- **M5 反向门控**：无对应功法的角色，其 Evade/护体模块**不存在** → 能力为 0（gate 硬断言，§13）。

## 6. 法宝配套战斗效果（§14.2 落地）

法宝不只是道具——**数值底 + 配套功法解锁的战斗模块**（与功法门控一致）：
- **(a) 数值底**：itemTier → pe 量级（已有 power 公式，器修 res:itemTier×40 主导）。无配套功法 = 裸数值底。
- **(d) 配套御器功法 → 解锁战斗模块**：
  - 御剑 = `PenFromResource`(Key=itemTier) ranged 攻（异化招）
  - 护宝罡 = OnDefend `AddFlatDR` + `ReflectDamage`（法宝护体 + 落宝反弹）
  - 落宝 = `Special`(luobao)（夺敌器，门控质变克制）
  - 万宝齐发 = `SumOfSet`(Key=itemTierSet, 挂带宽 cap)
- **器修 = 御宝流**：多 Special + SumOfSet + 落宝，其专门战斗方式。
- 法宝持有面（人人 vs 仅器系路）：本轮**沿用现有 itemTier 资源分布**（器修主导），不扩持有规则（YAGNI）。

## 7. Special handler 架构（唯一档逃逸口纪律，M3）

唯一档 = 有限算子集的扩展点，**最严纪律**（防红线 B.2 整数确定性破口）：

```csharp
public interface ISpecialModule {
    string HandlerId { get; }
    SpecialResult Apply(in CombatContext ctx);   // 纯整数; 输出 delta + 副作用(经 chokepoint)
}
public static class SpecialModuleRegistry {
    // handlerId → 单一 ISpecialModule (确定派发, 注册期校验无重复/无缺失)
}
```

**handler 纪律（gate M3 强制）**：
1. **整数纯度**：IL 浮点零扫描覆盖所有 handler（扩 `CultivationFloatScanTests`）。
2. **派发确定**：同 handlerId → 同函数（注册表冻结，注册期断言）。
3. **不读 daoHeart/innerDemon**（红线 R3 EffectivePower 解耦，对标 PowerEngine R3 guard）。
4. **不消费未声明随机**（本轮无 RNG；若 handler 需随机 → 拒，留 FULLSTRUCT 评审）。
5. **副作用经 chokepoint**：改资源走 `ApplyResource`、改关系走既有边 API（本轮签名机制多为削/夺资源，落 chokepoint）。

落宝/炸阵/夺舍/金身态/因果栈 各一 handler；未迁路的签名机制标 deferred（§10 覆盖账）。

## 8. determinism（M1/M2/M6）

- **无 RNG**：Evade 确定性（D-EVADE），R2 回合不掷骰 → 不引入运行期 `_cultRng` 消费 → **off 逐字节天然守 + ON 路 determinism 不受分支发散威胁**（auditor G5/M1 风险被 D-EVADE 从根上消除）。
- **模块结算确定（M2）**：模块列表用 `List`（顺序固定）；读资源用确定迭代（不靠 `Dictionary` 枚举序——若需遍历资源，排序键）；ratio 整数除，Amount2==0 视为 1。
- **同时结算无先手偏置（M6）**：双方 dmg 都读 pre-HP 算完再同时扣；镜像角色（同四维同 loadout）→ 必平 + Tiebreak(CharacterId)；交换 A/B 顺序结果对称。
- **ON 路逐字节**：构造 cultivation-ON 世界跑含 R2 战斗，同种子两跑 Chronicle 逐字节 + Clone 续跑等价（§13 gate M1，补 off gate 的 ON 盲区）。

## 11. 迁移计划（21 路招式 → 模块，分层落地顺序）

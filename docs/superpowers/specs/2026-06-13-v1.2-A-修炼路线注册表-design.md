## 1. 概述与目标

为江湖内核加**数据驱动、纯整数确定性、开闭可扩展**的修炼路线系统：每条路线一整套自有体系（命名各异的功法类目 + 战技 + 独立战力公式 + 独立战力曲线 + 修炼途径）。加新路线/功法/战技 = 追加数据，零改核心。

### 1.1 分阶段（本稿 = A.0 最薄竖切）

| 层 | 范围 | 文档 |
|---|---|---|
| **A.0（本稿，先建）** | 数据驱动引擎（`CultivationPathDef` + EffectOp 解释器 + PowerEngine per-path 整数战力 + RealmCurve + CounterMatrix + `CultivationState` 侧表） + **4 路代表完整接线**（剑/体/法/鬼，覆盖物理环克 + 精神 + 邪道） + 生成期定路/选功法战技 + **基础 realm**（单层 flatIndex + 阈值突破） + 战斗集成。Step 0 含全状态快照基础设施 + IL 浮点扫描。 | 本稿 |

### 1.2 用户硬要求对齐（A.0 即满足）

每路非换皮：4 路各有独有机制（剑修 本命剑/剑意/斩道；体修 横练罩门/血气/不灭金身；法修 法术库/元素相克；鬼修 煞值/昼夜/养鬼反噬）+ 命名各异功法类目 + 多具名功法（每类 ≥4）+ 战技池(≥5) + 独立战力公式 + 独立战力曲线 + 独立修炼途径。可扩展不锁死（数据驱动 / 克制独立装配 / 跨表 string key / **不污染四维生成期 Σ** / EffectOp 有限算子非自由文本）。

## 5. 数据模型（schema；4 路数据照搬每路深度设计文档）

```csharp
public sealed record CultivationPathDef(
    string PathId, string Name, string AttackDimension,        // physical|spirit|fate|economic
    IReadOnlyList<string> CounterKeys,                          // 仅声明自身键，克制矩阵独立
    IReadOnlyList<ResourceDef> Resources,                      // per-path 专属资源（剑意/血气/煞值…），不进四维
    PowerFormulaDef Power, RealmCurveDef Curve,                 // 独立公式 + 独立曲线
    IReadOnlyList<ArtCategoryDef> ArtCategories,               // 命名功法类目，每类 Arts≥4
    IReadOnlyList<CombatSkillDef> CombatSkills,                // 战技池 ≥5
    EntryGateDef EntryGate, SelectionRuleDef Selection,
    IReadOnlyDictionary<string,int>? Tuning);
// ResourceDef/ArtCategoryDef/ArtDef/CombatSkillDef/EntryGateDef/SelectionRuleDef/EffectOp 见每路深度设计文档 §1

public sealed class CultivationState {   // Character 可空字段；散修=null；off=null
    public string PathId { get; init; }
    public int RealmIndex { get; set; }                        // A.0 单层 flatIndex（A.1 投影双层，不存额外字段）
    public IReadOnlyList<string> ChosenArtIds { get; init; }
    public IReadOnlyList<string> ChosenSkillIds { get; init; }
    public Dictionary<string,int> Resources { get; }           // resourceKey→整数，经 ApplyResource chokepoint 钳制
    public Dictionary<string,int> Flags { get; }               // 0/1
    public CultivationState Clone();                            // 逐项深拷（R-NF2 命门，进 S0-a 快照对账）
}
```

## 6. PowerEngine — per-path 声明式整数战力

```
base = Σ term.weight × resolve(term.src, actor)   // src ∈ stat:* | realm(=RealmIndex) | sumArtPower | res:<key>
       （term.weightStepKey：所选功法/flag 可使某 term 权重 +N 阶）
raw  = clamp(base + 施加 modifiers, 0, ...)         // 整数定点 x*num/den，每步 clamp
final = (long)raw × Curve.realmMultipliers[RealmIndex] / 10   // 整数除，long 防溢出，clamp POWER_CAP
```
每路公式不同（剑修 Force×4+Insight×3+res:swordWill×3 凸曲线；体修 Constitution×3 无 Insight 平滑曲线；鬼修 spirit 维 + 煞值，昼夜 flag 调制）。禁浮点（S0-b 扫描守护）。

---

## 7. EffectOp — 有限算子集

功法/战技效果 = 引擎可解释的整数算子（非自由文本 → 确定性可复现）：`AddTermWeightStep / AddResourceCap / AddResource / AddFlatDR / SetFlag / AddPenInteger / ScalarMul / AddCounterAdj / GrantPassive`。被动功法装配期施加；主动战技 OnUse 战斗期施加，消耗 Cost 资源，经 `IWorldMutator`（属性仍只过 `StatBlock.Apply`）。`Name/Note` 是 flavor 不进数值。新功法/战技 = 新数据；仅全新语义才扩 `EffectOpKind`（明确边界）。**A.0 先盘点 4 路所有 effects 收敛出最小封闭算子集再实现解释器**（先写算子清单测试）。

---

## 8. CounterMatrix — 克制（独立装配）

`data` 边表 `{attacker, defender, coef:整数%, guard:谓词?}`。A.0 接 4 路所涉边（物理环克 剑>体、体>法近身、法>体远程；精神 魂/鬼绕物防；邪正 佛克鬼——A.0 含鬼修故含佛克鬼边 + 雷/纯阳；guard 读 distance/is_night）。`effective = Evaluate(atk) × coef / 100`（整数）。路线只声明 `CounterKeys`，不硬编码对手（改克制不动路线）。

---

## 9. Realm（A.0 基础单层）

`RealmIndex` 0..Nₚ（= 该路 `realmMultipliers` 长度）。战力经 Curve 放大。`修炼` 动作累加一个本路修为资源 → 达 `RealmThreshold[next]` → `RealmBreakthrough`（RealmIndex++，封顶）。**A.0 突破确定性**（达阈值即突破，不掷随机）。**大小境界双层（三轴解耦 flatIndex/(Major,Sub)/UnifiedTier）+ 瓶颈/劫/寿元 = A.1**（A2-FINAL，零改本稿曲线与战力公式）。

## 10. 生成期定路 + 选功法战技（确定性，cultRng/genRng）

`WorldFactory.Spawn`（on）在既有 genRng 消费**之后**（append-only）：生成 `Persona.Tags` → 评估 4 路 `EntryGate`（纯查询 `CanEnter`）→ VariedSelector 加权抽取定 `PathId` → `SelectionRule` 各类目选 1 功法 + N 战技（tier ≤ realm 闸，NPC 确定性贪心）→ 初始化 `CultivationState`（resources 经 ApplyResource，realm=0）→ 产 `PathEntered`。全经 cultRng=Split(5)/genRng。

## 13. 需求规格（EARS，纠错后）

- **R-A-F1**（凡）：内核 SHALL 从 `IPathSource` 加载 4 条修炼路线定义，每条含独立资源/战力公式/战力曲线/命名功法类目(≥3 类每类≥4)/战技池(≥5)。
- **R-A-F2**（当）：WHEN 生成角色（cultivation-on），内核 SHALL 经 EntryGate 定路、各类目选 1 功法+N 战技、初始化 CultivationState、产 `PathEntered`。
- **R-A-F3**（凡）：内核 SHALL 用每路各自的声明式整数公式×境界曲线计算战力。
- **R-A-F4**（当）：WHEN 两修士切磋，内核 SHALL 以 per-path 战力 × 克制系数(CounterMatrix,guard 上下文)结算。
- **R-A-F5**（当）：WHEN 修为达 RealmThreshold，内核 SHALL 提升 RealmIndex(封顶)并产 `RealmBreakthrough`。
- **R-A-F6**（凡）：功法/战技效果 SHALL 表为有限 EffectOp 算子集，引擎确定性解释。
- **R-A-NF1 可扩展**（凡）：加新路线 SHALL 仅追加数据，零改引擎核心。
- **R-A-NF2 四维契约**（凡）：内核 SHALL NOT 旁路 `StatBlock.Apply` chokepoint，且 SHALL NOT 改 `StatGenerator` **生成期** Σ=80（**运行期 Σ 非 80，不断言**——C1）。
- **R-A-NF3 确定性**（在…期间）：WHILE cultivation-on 且全 RuleBrain，内核 SHALL 对同 (种子,配置) 逐字节一致（经 S0-a 全状态快照对账，非仅 Chronicle），快照续跑亦一致。
- **R-A-NF4 安全网**（在…期间）：WHILE cultivation-off，内核 SHALL 产生与 v1.0 完全一致轨迹（38 测试逐字节，cultRng=Split(5) 不构造，Split(1..4) 子流 GetState 不变）。
- **R-A-NF5 整数纯度**（凡）：`Jianghu.Cultivation` SHALL 不含浮点（S0-b 新建 IL 扫描验证）。
- **R-A-NF6 资源边界**（凡）：每 per-path 资源 SHALL 恒在 [Min,Cap] 内（经 ApplyResource chokepoint）。

    IReadOnlyList<string> SituationalTags,                      // 属性/形态 tag（R2：melee/ranged/spirit_attack/evil/fire…，非对手 PathId）

## 6. PowerEngine — per-path 声明式整数战力

```
BaseSum    = Σ term.weight × resolve(term.src, actor)   // src ∈ stat:* | realm | sumArtPower | res:<key> | derived:<key>
             （禁 ×0 term R6；禁 src=daoHeart/innerDemon R3；term.weightStepKey：功法/flag 升某 term 权重）
Modified   = clamp(BaseSum 顺序过 Modifiers, 0, ...)     // 有序整数定点 x*num/den，每步 clamp；新 ModKind 登记 L1（R5）
final      = (long)Modified × Curve.realmMultipliers[RealmIndex] / 10   // 整数除，long 防溢，clamp POWER_CAP
                + postMul（flag 条件乘子，整数 num/den）
```
每路公式不同（剑修 `Force×4+Insight×3+res:swordWill×3` 凸曲线；体修 `Constitution×3` 平滑；儒修 `Insight×4+res:haoran×3+realm×5+...`**不含 daoHeart**）。**P0 当量校验**：四维=20(Σ80) 基线下各路同 UT 战力量级可比（标定属 A.3）。禁浮点（S0-b 守护）。

## 7. EffectOp — 有限算子集（核心 + L1 扩展）

功法/战技效果 = 整数算子。**核心 `EffectOpKind`**（盘点 21 路收敛）：`AddTermWeightStep / AddResourceCap / AddResource / AddFlatDR / SetFlag / AddPenInteger / ScalarMul / AddSituationalAdj / GrantPassive / Cost`。被动功法装配期施加；主动战技 OnUse 战斗期施加，消耗 Cost 资源，经 `IWorldMutator`（属性仍只过 `StatBlock.Apply`）。`Name/Note`=flavor。**新语义算子（per-path 必需的，如血修归零、音修范围乘）= 新 `EffectOpKind`/`ModKind`，标 L1（R5）**，纯整数单测。先写「21 路算子清单」测试圈定边界，再实现解释器。

## 8. SituationalEdge — 软情境战斗（替代 CounterMatrix，R2）

```csharp
public sealed record SituationalEdge(
    string Axis,            // element|range|terrain|time|form|alignment （战斗轴，非 PathId）
    string WhenPred,        // 谓词 DSL：读 attacker/defender 的 SituationalTags + 环境（distance/is_night/terrain/qiDensity）
    int CoefPct);           // 整数 % adj（正=增益攻方，负=减益）
```
- **零 PathId**：边只认 tag + 环境（如 `attacker.fire & defender.tag:ice → +15`；`defender.tag:undead_construct & attacker.axis:spirit_attack → -100`(死物免精神)；`is_night & attacker.tag:ghost → +10`；`distance:far & attacker.tag:melee_brute → -20`）。剑与体之间**没有边**。
- 结算：`adj = Σ 命中边 CoefPct`，**clamp ±(P0/4)**（情境只修正，realm×战力主导，不翻盘）；`effective = PowerEngine × realmMul/10 × (100+adj)/100`（整数）。
- 役使三态（活体/死物/寄生）、邪正链（anti_evil tag 克 evil tag）、斩首脆弱（指挥链 tag）、绕物防（spirit_attack axis）**全部表为 tag+谓词边**，不写路线身份。路线只声明 `SituationalTags`，改边不动路线。

## 9. Realm（A.0 基础单层）

`RealmIndex` 0..Nₚ（=该路 `realmMultipliers` 长度）。`修炼` 动作累加本路修为资源 → 达 `RealmThreshold[next]` → `RealmBreakthrough`（RealmIndex++ 封顶）。**A.0 突破确定性**（达阈即突破，不掷随机 → 无运行期 cultRng 消费时可不升 World 字段；若设计要随机择优则升字段 R1）。**大小境界双层 + 瓶颈/劫/寿元 = A.1**（零改本稿曲线公式）。
> **数据校验（backlog2-M4）**：每路 `RealmCurveDef` 的 `realmMultipliers` 长度 == `UnifiedTierOf` 长度 == 境界称谓行数（因果路原案 8 vs 7 不匹配 → 校验测试拦截）。

## 1. 概述与目标

为江湖模拟内核加入**武侠+仙侠双主题的修炼路线系统**：每个角色循一条"修"（剑修/体修/法修/鬼修…），每条路线是**一整套自有体系**——独立的功法类目（命名各异）、独立的战力衡量公式、独立的战力曲线、独立的修炼途径、专属战技。系统**数据驱动、纯整数确定性、开闭可扩展**：加新路线 = 追加一份 JSON，零改核心代码。

### 1.1 分阶段（本规格 = A.0）

| 阶段 | 范围 | 验收 |
|---|---|---|
| **A.0（本规格）** | 数据驱动引擎（CultivationPathDef + EffectOp 解释器 + PowerFormula 求值 + RealmCurve + CounterMatrix + CultivationState） + **12 路完整数据接线** + 生成期定路/选功法战技 + realm 突破 + 战斗集成 | 12 路角色生成、战斗按 per-path 公式+克制+境界结算、编年史可读、cultivation-on 确定性逐字节复现、cultivation-off 时 v1.0 的 38 测试逐字节不变；加一条 mock 路线零改引擎 |
| **A.1** | 其余 9 路 + 21 catalog 补齐、转职/觉醒/双修、反噬 RiskModifier、realm-寿元耦合、克制系数标定、随机奇遇/丹药 | — |
| **B** | 戏剧引擎（恩怨/复仇弧），见姊妹 spec | — |

### 1.2 用户要求对齐（硬约束，本设计逐条满足）

1. 每条路径一整套体系，**非换皮**（各有独有机制：剑修 本命剑/剑意/斩道；体修 横练罩门/血气/不灭金身；鬼修 煞值/昼夜/养鬼反噬…）。
2. 功法类目**命名随路径变**（剑修=剑法/心法/身法/剑意；法修=法术/符印/灵力运转；体修=横练功/拳脚功）。
3. 每类**多个具名功法**（种数不能少，每类 ≥4），每人每类选一。
4. **特殊战技池**（≥5），每人选若干。
5. **每路独立战力衡量公式**（剑修 武力×4+悟性×3+剑意×3；体修 根骨×3 无悟性）。
6. **每路独立战力曲线**（剑修凸爆发 `[10,14,20,30,46,70,108,170,270]`；体修平滑 `[10,13,17,22,28,35,43,52,62,73]`）。
7. **可扩展不锁死**：数据驱动、克制独立装配、跨表 string key、不污染四维定和、effects 用有限算子集而非自由文本。

## 2. 已锁定约束（决策记录）

| 决策 | 结果 | 理由 |

## 3. 架构

新命名空间 `Jianghu.Cultivation`，落 `Jianghu.Core`（netstandard2.1，禁浮点）。被改既有文件**最小化纯加法**：
- `Model/Character.cs`：加可空字段 `CultivationState? Cultivation`（Clone 纯加拷贝）。**不动四维 StatBlock 与 Σ 契约**。
- `Model/Persona.cs`：加 `IReadOnlyList<string> Tags`（灵根/资质/体质，驱动 entry_gate）。
- `Sim/WorldFactory.cs`：`CreateInitial` 加 `bool cultivation = false` 开关（**默认 false** → 既有 caller 与 v1.0 的 38 测试签名/轨迹逐字节不变；CLI 显式传 `true` 启用）；on 时 `Spawn` 末尾（既有 genRng 消费**之后**，append-only）定路+选功法战技。
- `Actions/SparAction.cs`：`Power` 改为——**无 PathId → 旧公式**（保 off 轨迹）；**有 PathId → `PowerEngine.Evaluate(actor)` per-path 公式 × realm 曲线 × 克制系数**（整数）。
- `Events/DomainEvent.cs` / `Events/Chronicle.cs` / `Config/LimitsConfig.cs`：纯加 case / 字段。

```
Jianghu.Cultivation/
  CultivationPathDef.cs   # 声明式路线定义（record 树）
  EffectOp.cs             # 有限算子集 + 解释器契约
  PowerEngine.cs          # per-path 整数战力求值（term-list + modifiers + curve）
  RealmCurve.cs           # 境界倍率 + 调制器（双包络/昼夜）
  CounterMatrix.cs        # 克制矩阵（独立装配，边带 guard）
  CultivationState.cs     # 角色侧运行态（resources/flags/chosen/roster）
  IPathSource.cs          # 路线数据源端口（A.0 = CodePathSource 常量库）
  PathRegistry.cs         # 加载+索引（按 entry_gate/counterKey）
  PathAssigner.cs         # 生成期定路 + 选功法战技（确定性加权）
  paths/*.cs              # 12 路数据（CodePathSource）
```

## 4. 数据模型（schema 摘要；全文见每路深度设计文档 §1）

```csharp
public sealed record CultivationPathDef(
    string PathId, string Name, string AttackDimension,        // physical|spirit|fate|economic
    IReadOnlyList<string> CounterKeys,                          // 仅声明自身键，克制矩阵独立
    IReadOnlyList<ResourceDef> Resources,                      // per-path 专属资源（不进四维）
    PowerFormulaDef Power, RealmCurveDef Curve,                 // 独立公式 + 独立曲线
    IReadOnlyList<ArtCategoryDef> ArtCategories,               // 命名功法类目
    IReadOnlyList<CombatSkillDef> CombatSkills,                // 战技池
    EntryGateDef EntryGate, SelectionRuleDef Selection,
    IReadOnlyDictionary<string,int>? Tuning);                  // 平衡旋钮，整数

public sealed record ResourceDef(string Key, int Min, int Max, string? CapFormula,
    int Initial, string? InitialExprKey, bool ResetEachSpar);
public sealed record ArtCategoryDef(string CategoryKey, string DisplayName, string Role,
    int PickMin, int PickMax, IReadOnlyList<ArtDef> Arts);     // Arts ≥4
public sealed record ArtDef(string ArtId, string Name, int Tier, int Power,
    IReadOnlyList<EffectOp> Effects, IReadOnlyDictionary<string,int>? Gates);
public sealed record CombatSkillDef(string SkillId, string Name, int Tier, CostDef Cost,
    IReadOnlyList<EffectOp> OnUse, IReadOnlyDictionary<string,int>? Gates, string Family); // ≥5
public sealed record EntryGateDef(IReadOnlyDictionary<string,int> StatThresholds,
    IReadOnlyList<string> QualityTagsAnyOf, IReadOnlyList<string> Excludes, int Rarity);
public sealed record SelectionRuleDef(int SkillPickMin, int SkillPickMax,
    string TierGateExpr, IReadOnlyList<string> MandatoryFamilies, string GreedyKey);
```

**角色侧运行态（不动四维）**：
```csharp
public sealed class CultivationState {   // Character 上可空字段；散修=null
    public string PathId { get; init; }
    public int RealmIndex { get; set; }
    public IReadOnlyList<string> ChosenArtIds { get; init; }
    public IReadOnlyList<string> ChosenSkillIds { get; init; }
    public Dictionary<string,int> Resources { get; }   // resourceKey→整数，经 ApplyResource chokepoint 钳制
    public Dictionary<string,int> Flags { get; }        // arrayed/openedSword/isNight→0/1
    public IReadOnlyList<string> RosterIds { get; init; } // 灵兽/法宝/鬼兵外部条目（A.0 可空）
    public CultivationState Clone();
}
```
- 资源变更只经 `ApplyResource(key, delta, def)` 单点钳制到 `[Min, CapFormula(realm)]`（对齐 `StatBlock.Apply` 纪律）。
- 12 路的完整 `Resources/Power/Curve/ArtCategories/CombatSkills/EntryGate` 数据 = 每路深度设计文档逐路全文，A.0 实现期照搬为 `CodePathSource` 常量（developer 按文档落数据，引擎不含路线特例）。

## 5. PowerEngine — per-path 声明式整数战力求值

每路 `PowerFormulaDef` = `terms[] + modifiers[] + finalScale`。求值（全整数）：
```
base = Σ term.weight × resolve(term.src, actor)      // src ∈ stat:Force/Insight/… | realm | sumArtPower | res:<key>
       （term 可带 weightStepKey：所选功法/开光 flag 可使某 term 权重 +N 阶）
mod  = 依次施加 modifiers（条件乘子/加项，整数定点 x*num/den）
raw  = clamp(base + mod, 0, …)
final = raw × Curve.realmMultipliers[RealmIndex] / 10   // 整数除；再过 Curve.postMul/envelopes 调制
```
- `resolve` 把 `stat:*` 读 StatBlock、`realm` 读 RealmIndex、`res:*` 读 CultivationState.Resources、`sumArtPower` = 所选功法 tier→power 之和。
- **每路公式不同**（剑修 Force×4+Insight×3+res:swordWill×3 凸曲线；体修 Constitution×3 无 Insight 平滑曲线；魂修 spirit 维绕物防…）——全部数据，引擎统一解释。
- 禁浮点：所有除法整数除、定点 `x*num/den`、long 防溢出。

## 6. EffectOp — 有限算子集（功法/战技效果的确定性落点）

功法/战技效果**不写自由文本**，而是一组引擎可解释的整数算子，保确定性可复现：
```
enum EffectOpKind { AddTermWeightStep, AddResourceCap, AddResource, AddFlatDR, SetFlag,
                    AddPenInteger, ScalarMul, AddCounterAdj, GrantPassive,
                    AdjustRelationEdge, ApplyStatDelta }
record EffectOp(EffectOpKind Kind, string? TargetKey, int A, int B, string? Note);
```
- 被动功法 → 装配期施加（改 PowerFormula 项权重/资源上限/常驻减伤/flag）。
- 主动战技 `OnUse` → 战斗结算期施加（消耗 Cost 资源 + 造伤/控制/护体），经 `IWorldMutator` 落地（属性仍只过 `StatBlock.Apply`，关系只过 `Relations.Adjust`，保 DomainEvent 单源）。
- `Note`/`Name` 是 flavor 文本，**不进数值路径**。
- A.0 实现一个**封闭的算子解释器**；新功法/战技 = 新数据（算子组合），零改引擎；仅当需要全新算子语义才扩 `EffectOpKind`（明确边界）。

## 7. CounterMatrix — 克制（独立装配，三正交轴）

`data` 常量边表：`{attacker:counterKey, defender:counterKey, coef:整数%, guard:谓词?}`。三轴（物理环克 / 精神绕物防 / 气运因果泛克）+ 资源依附层，详见 registry 研读文档"克制关系网"。
- 战斗结算：`effective = PowerEngine.Evaluate(atk) × counterCoef(atk.path, def.path, guardCtx) / 100`（整数，每步 clamp）。
- `guard` 谓词读上下文变量（distance/arrayed_flag/tier_gap/is_night），**做成边属性而非写死单向**（防僵化）。
- 路线本体只声明 `CounterKeys`，**不硬编码对手** → 改克制不动路线（开闭）。

## 8. Realm 境界与突破（A.0 确定性）

- `RealmIndex` 0..N（每路 N 可不同；统一武侠↔仙侠刻度）。战力经 `Curve` 放大。
- `修炼` 动作（既有）：按本路 `cultivationRoute` 推进——A.0 简化为累加一个本路修为资源（如剑修 zhanDaoMiles、体修 血气/根骨日课）到 `RealmThreshold[next]` → `RealmBreakthrough`（RealmIndex++，封顶）。**A.0 突破确定性**（达阈值即突破，不掷随机品阶；随机奇遇/丹药突破留 A.1）。
- **realm 不延寿**（A.0）：保 v1.0 寿命带/人口涌现不受扰（realm-寿元耦合是已识别 gap，A.1 专门规约）。

## 9. 生成期定路 + 选功法战技（`PathAssigner`，确定性）

`WorldFactory.Spawn`（cultivation-on）在既有 genRng 消费**之后**（append-only）：
1. 生成 `Persona.Tags`（灵根/资质/体质，VariedSelector 从 tag 池抽，用 genRng）。
2. 评估 12 路 `EntryGate`（stat 阈值 + tag anyOf + excludes，纯查询 `CanEnter`）→ 合格集合。
3. `VariedSelector` 加权抽取定 `PathId`（权重 = Rarity × 匹配度）。
4. 按 `SelectionRule` 各类目选 1 功法 + N 战技（tier ≤ realm 闸；NPC 确定性贪心键，按 archetype/stat 排序）。
5. 初始化 `CultivationState`（resources/flags/realm=0），产 `PathEntered`。
- 全经 genRng（无新 RNG 流）→ cultivation-on 是新确定性轨迹；cultivation-off 不走此路径 → 旧轨迹不变。

## 10. 与 v1.0 集成 · 确定性

- **战斗**：`SparAction.Power` 分流（见 §3）。off/散修 → 旧公式（38 测试不变）；on → PowerEngine × curve × counter。`DuelResolved.Margin` 反映之。
- **CultivationState 挂 Character**（可空字段，纯加）；`Character.Clone` 加 `Cultivation?.Clone()`（R-NF2 续跑命门——逐项深拷 resources/flags/chosen/roster/realm）。
- **DomainEvent 单源**：新增 `PathEntered/RealmBreakthrough`（+ A.1 转职觉醒）。`Project`/`Chronicle.Append` 纯加 case。
- **RNG**：无新 Split 流（A.0 全经 genRng + 确定性运行期）。保留 `_domainRng`（reserved）。Split 编号：A.0 不占；戏剧(B) 用 Split(5)（其 spec 已定）。
- **确定性（关键）**：
  - cultivation-**off**：`WorldFactory.CreateInitial(cultivation:false)` → 路线系统完全不参与（SparAction 走旧公式、Spawn 不定路）→ **v1.0 的 38 测试逐字节不变**（安全网）。
  - cultivation-**on**：新 INV-DET-Cult-1（同种子同编年史）+ INV-DET-Cult-2（含修士的 World.Clone 续跑逐字节一致）。
  - 全整数禁浮点；新增 IL 浮点扫描覆盖 `Jianghu.Cultivation`（CorePurityTests 现无此能力，须加 — 同戏剧 spec 已识别）。

## 12. 需求规格（EARS，带 ID — 供 writing-plans 追溯）

- **R-A-F1**（凡）：内核 SHALL 从 `IPathSource` 加载 12 条修炼路线定义，每条含独立的资源/战力公式/战力曲线/命名功法类目/战技池。
- **R-A-F2**（当）：WHEN 生成一名角色（cultivation-on），内核 SHALL 按其 stat+tags 经 EntryGate 定一条路线、各类目选 1 功法 + N 战技、初始化 CultivationState，并产 `PathEntered`。
- **R-A-F3**（凡）：内核 SHALL 用每条路线**各自的**声明式整数公式 + 境界曲线计算其战力（不同路线公式不同）。
- **R-A-F4**（当）：WHEN 两名修士切磋，内核 SHALL 以"per-path 战力 × 克制系数（CounterMatrix，按 guard 上下文）"结算胜负。
- **R-A-F5**（当）：WHEN 角色修为达 `RealmThreshold[next]`，内核 SHALL 提升其 RealmIndex（封顶）并产 `RealmBreakthrough`。
- **R-A-F6**（凡）：功法/战技效果 SHALL 表为有限 EffectOp 算子集（非自由文本），由引擎确定性解释。
- **R-A-NF1 可扩展**（凡）：新增一条路线 SHALL 仅追加数据（path def + counter 边），**零改引擎核心代码**。
- **R-A-NF2 不污染四维**（凡）：路线系统 SHALL NOT 改变四维定和契约（任意时刻 `Σstat=80`、单项 ≤cap）。
- **R-A-NF3 确定性**（在…期间）：WHILE cultivation-on 且全 RuleBrain，内核 SHALL 对同 `(种子,配置)` 产生逐字节一致轨迹；WHEN 快照续跑亦一致。
- **R-A-NF4 回归安全网**（在…期间）：WHILE cultivation-off，内核 SHALL 产生与 v1.0 完全一致的轨迹（38 测试逐字节不变）。
- **R-A-NF5 整数纯度**（凡）：`Jianghu.Cultivation` SHALL 不含任何浮点（IL 扫描验证）。
- **R-A-NF6 资源边界**（凡）：每个 per-path 资源 SHALL 恒在 `[Min, Cap(realm)]` 内（经 ApplyResource chokepoint）。

## 14. 风险与 gaps（实现期收敛）

- **平衡（最大）**：12 路战力当量/曲线/克制系数全是初值，易某路碾压。缓解：集中 `Tuning`/`LimitsConfig` 旋钮 + 跨路对拍测试 + 建议独立标定阶段（类比 calibration）。详见每路深度设计文档"balanceNotes"/"gaps"。
- **realm 跨域当量**：武侠武评 vs 仙侠境界同 index 战力当量未定（gap）→ A.0 用统一曲线占位，A.1 标定。
- **EffectOp 解释器复杂度**：算子集需覆盖 12 路所有功法/战技效果；A.0 须先盘点 12 路 effects 收敛出最小封闭算子集，再实现解释器（先写算子清单测试）。
- **确定性面扩大**：CultivationState 多资源/flags 续跑敏感 → Clone 逐项对账 + INV-DET-Cult-2 专测。
- **内容工作量**：12 路 × (≥3 类目×≥4 功法 + ≥5 战技) ≈ 200+ 具名条目落数据；developer 照搬深度设计文档，按本世界观收敛取舍。

# 修炼 A.0（21 路引擎 · 软情境 · 确定性）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. **所有 implementer/reviewer subagent 用 Opus 4.8。**

**Goal:** 给 `Jianghu.Core` 加数据驱动、纯整数确定性、开闭可扩展的修炼系统：21 路全部入册（不舍弃），引擎一次建成，per-path 独立战力公式/曲线/功法/战技，软情境战斗（无硬 path 克制），基础单层 realm。cultivation-off（默认）→ v1.0 的 38 测试逐字节不变。

**Architecture:** 新命名空间 `Jianghu.Cultivation`（netstandard2.1，禁浮点）。既有文件最小纯加法（off 默认逐字节）。引擎 = `CultivationPathDef`（数据）→ `PowerEngine`（terms→BaseSum→Modifiers→Curve 整数求值）+ `RealmCurve` + `EffectOp`（有限算子解释器）+ `SituationalResolver`（Axis+WhenPred 软情境，零 PathId，adj clamp±P0/4）+ `CultivationState`（角色侧表，深拷）。生成期 `PathAssigner` 定路选功法。战斗期 `SparAction` 分流。21 路数据各一文件（`CodePathSource`），照搬深度设计文档。

**Tech Stack:** C# / .NET（`Jianghu.Core` netstandard2.1，`Jianghu.Core.Tests` xUnit，`Jianghu.Cli` net8.0）。Pcg32 PRNG。`System.Reflection.Metadata` 做 IL 浮点扫描。

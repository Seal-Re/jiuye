## 数据驱动可扩展注册表设计

# 数据驱动注册表 schema 设计（开闭·解耦·与 v1.0 确定性整数核兼容）

## 0. 设计契约（铁律对齐）

四个边界条件必须同时满足，任何 schema 字段不达标即不进模型：
1. **可映射整数确定性**: 所有判定=整数阈值比较 + VariedSelector 加权抽取 + cooldown 计数 + 克制矩阵整数系数。**禁浮点参与世界状态**（对齐 StatGenerator.cs §4.1 纯整数采样、Relations 钳制 [-100,100]）。
2. **开闭原则**: 加新路线/背景/桥段 = 往 JSON 模板池追加一条记录，**零改核心代码**。所有跨路线引用走 string key（PathId/标签），禁硬编码 enum 互相引用。
3. **字段纪律**（design §14: "有字段无机制不进模型"）: 新维度（道德/气运/寿元/神识带宽/心魔值/煞气/血煞/毒值/羁绊度/心境道心）仅在对应路线机制落地时才入数据模型，作为**可选按需挂载维度**，不污染 v1.0 四维定和约束。
4. **DomainEvent 单源投影**: 入门/突破/转职/觉醒/恩怨推进全产 DomainEvent，经单源投影确定性可重放（对齐 design §7.1）。

## 1. 目录结构（对应 design §13 `data/`）

```
data/
  cultivation_paths.json     # 路线模板池（≥21 条，对位 §6.2 SkillTemplate/SectTemplate 扩展）
  path_branches.json         # 转职分支（与主路线松耦合，可独立增删）
  counter_matrix.json        # 克制矩阵（独立装配，改克制不动路线本体）
  resource_pools.json        # 具名资源池（功法/材料/天材地宝/本命器/符兽傀曲蛊卜法）
  origins.json               # 背景标签池（origin_bonus 阈值偏移 + 预注入冤孽种子）
  drama_arcs.json            # 戏剧弧状态机模板（15 条）
  storylets.json             # 桥段 storylet（前置+效果，QBN 任意序）
  enmity_templates.json      # 恩怨链模板（复仇/夺嫡/身世/吞并/对赌）
  risk_modifiers.json        # 反噬组件（可插拔，挂在路线声明上）
```

## 2. 核心 schema：CultivationPath（路线模板）

```jsonc
{
  "id": "sword_cultivator",          // string 主键，跨表唯一引用键（禁 enum 互引）
  "name": "剑修/剑仙",
  "domain": "orthodox_offense",
  "attack_dimension": "physical",    // physical|spirit|fate —— 正交克制图分层键
  "entry_gate": {
    "stat_thresholds": { "Insight": 18, "Force": 15 },   // 复用 StatKind 四维整数阈值
    "quality_tags_anyOf": ["jin_linggen", "sword_bone", "sword_heart"],
    "bg_tags_anyOf": ["sword_tomb_origin", "shushan_lineage"],
    "trigger_events_anyOf": ["pickup_broken_sword", "sword_spirit_recognition", "kill_to_awaken_intent"],
    "moral_threshold": null,         // 仅儒修/佛修等少数路线非 null（隐藏道德维度门）
    "excludes": ["pure_ghost", "thunder_cultivator"]   // 防非法兼修组合
  },
  "origin_bonus": [ { "bg_tag": "sword_tomb_origin", "stat_offset": -3 } ],  // 背景=阈值偏移量，单列不写死进路线
  "resource_pool_refs": ["sword_arts", "life_sword", "sword_temper_mat"],    // 引 resource_pools.json
  "realm_anchor": "qi→foundation→golden→nascent→spirit",  // 统一 realm_index 锚（炼气0→筑基1→…）
  "growth_curve": "leap",            // linear(吃资源/熟练度) | leap(领悟跃迁) —— 武侠内功层数与仙侠境界同表对齐
  "secondary_index": null,           // 辅助职(丹/器/阵/符/傀/命/音/蛊)的"品阶"与"修为"半解耦，用独立 index
  "hidden_axis": null,               // 偏门系专属隐藏计量(心魔/煞气/血煞/毒值/气运)，一路线一轴，单调累积+cooldown衰减
  "counter_keys": ["sword"],         // 在 counter_matrix 中的键，本体不写死对手
  "branch_refs": ["sword_slaughter", "myriad_swords", "sword_array_dual", "guardian_sword", "sword_slave_demon"],
  "risk_modifier_refs": [],          // 偏门系挂 RiskModifier（剑修无）
  "exemplars": ["诛仙青云御剑", "剑来陈平安", "雪中邓太阿"]   // 仅 FlavorText 池，绝不回写确定性 DomainEvent
}
```

**解耦要点**: `entry_gate` 编译成纯查询 `CanEnter(character, gate) -> bool`（对位 design §8 IAction.CanExecute 纯查询范式）。命中多条路线→VariedSelector 加权抽取（权重=稀有度×匹配度×机缘指定），不内藏 RNG（接收注入 IRandom）。

## 3. 克制矩阵 CounterMatrix（独立装配，避免硬耦合·边带 guard 防僵化）

```jsonc
{
  "edges": [
    { "attacker": "sword", "defender": "body", "coef": 130, "guard": null },        // 整数百分比，130=+30%
    { "attacker": "body",  "defender": "law_mage", "coef": 130, "guard": { "var": "distance", "op": "<=", "val": 1 } },  // 近身才克
    { "attacker": "law_mage", "defender": "body", "coef": 130, "guard": { "var": "distance", "op": ">", "val": 1 } },     // 远程反克 → 距离决定方向的石头剪刀布
    { "attacker": "array", "defender": "sword", "coef": 140, "guard": { "var": "arrayed_flag", "op": "==", "val": 1 } },  // 已布阵才克
    { "attacker": "artifice", "defender": "sword", "coef": 125, "guard": { "var": "tier_gap", "op": ">=", "val": 1 } },   // 法宝品阶差≥1才压
    { "attacker": "thunder", "defender": "evil",  "coef": 160, "guard": null },      // 雷克邪大倍率
    { "attacker": "fate", "defender": "*", "coef": 150, "guard": null },             // 因果天敌位泛克
    { "attacker": "fate", "defender": "fate", "coef": 100, "guard": null }           // 同类法则内部制衡（无加成）
  ],
  "self_risk": [
    { "path": "thunder", "trigger": "cast", "guard": { "var": "Constitution", "op": "<", "val": 18 }, "penalty": "self_backlash" },
    { "path": "soul",    "trigger": "soul_strike_fail", "penalty": "shenshi_rebound" },
    { "path": "fate",    "trigger": "any_use", "penalty": "heaven_punish_luck_drain" }
  ]
}
```

**三条结构性克制轴**（防退化为单维石头剪刀布）：
- **物理 physical**: 剑>体、体>法(近身)、法>体(远程)、阵>剑/体(已布阵)、器>剑(品阶差)、法>阵 —— 环状制衡，方向由 guard 上下文变量翻转（distance/arrayed_flag/tier_gap），**guard 做成边属性而非写死单向**。
- **精神 spirit 正交轴**: 魂修/音修绕物理防御（attack_dimension=spirit），单独一张克制图。
- **概率/气运 fate 层**: 命修/因果修先手情报压制+夺运，正交于实体克制，被高气运者反弹自克。
- **资源/依附层 relation_economic**: 丹/器/符对战斗路线'供给上位'，单列与战斗克制分离（防把经济关系误建模为攻防边）。

## 4. 转职/觉醒五阶段生命线（QBN storylet 式，三种转职形态）

```
散修(Drifter,无PathId) → 入门(Entry,绑首条路线) → 主修(Mainline,沿realm_index) → 分支(Branch,同路线特化) → 转职(Transmute/Awaken/Dual)
```

转职三形态统一走 storylet（前置 AND 触发 → 效果改 PathId/Realm/资源）：
- **觉醒 Awaken**: 隐藏血脉/体质/灵根触发（隐藏→线索→揭露→剧变多实例范式），门控=隐藏标签+触发机缘（濒死/秘境/血脉之物）。可把废灵根逆转天灵根（废柴逆袭）。
- **转修 Transmute**: 转化原路线改投新路线，代价=原进度部分清零或保留为底蕴，门控=新路线 entry_gate + 旧路线高境界前置。
- **双修 Dual**: 叠加第二条路线，门控=两路线资质均达标 + 兼容性矩阵（`excludes` 字段，如纯阳雷修与阴煞鬼修不可兼）。

正邪分叉（血符/尸傀/夺运/魔音/毒尊）统一走'道德阈值下滑→触发正邪对抗变体+天劫/正道围剿加重'，复用同一套 storylet 而非每路线写死。

## 5. 反噬作为可插拔 RiskModifier（开闭关键·不入核心循环）

```jsonc
{
  "id": "demon_heart_backlash",
  "trigger": "breakthrough|cast|kill",         // 触发时机枚举
  "probability_permille": 80,                  // 整数千分位（禁浮点）
  "negative_delta": { "Constitution": -2, "hidden:心魔值": 5 },
  "event": "qi_deviation|backlash|sudden_death|heart_demon",  // 走火入魔=隐藏血脉式"剧变"事件可复用
  "cooldown": 30,                              // 衰减/恢复 cooldown
  "clear_condition": "buddha_redemption|moral_above_threshold"
}
```

挂在路线 `risk_modifier_refs`，加新路线/新反噬桥段不改核心。RimWorld/CK3 治理对位: 每类觉醒/奇遇/渡劫事件按 good/bad/threat/quest 分类 + cooldown_days + on/off 预算 + on_action 钩子 + cooldown_against_recipient（防同一角色被反复夺运/反复渡劫）。

## 6. 内核字段扩展（最小侵入·与 v1.0 兼容）

| 扩展点 | 建议 | 兼容性保证 |
|---|---|---|
| `Persona` | 增 `IReadOnlyList<string> Tags`（灵根/资质/体质/背景标签，驱动 entry_gate；可 VariedSelector 生成）；`ArchetypeKind` 由 enum 扩为外挂路线标签 `PathId?` | 纯数据追加，不破坏既有 record |
| 新 record | `PathId(string Value)` + `CultivationPath(...)` 全从 JSON 加载 | 无核心耦合 |
| `Character` | 增 `PathId? Path` + `int Realm`(境界序号) + `IReadOnlyDictionary<string,long> PathResources`(神识/才气/剑意/气运 long 防溢出) | 可空字段，散修=null |
| 第五维隐藏资质 | 建模为 Persona.Tags 分级标签（剑道资质/神识资质/灵根纯度），**不扩 StatKind** | 避免污染四维定和(sum=80/cap=30/min=5) |
| 新 DomainEvent | `PathEntered/RealmBreakthrough/PathTransmuted/BloodlineAwakened` 经单源投影 | 对齐 DomainEvent.cs 既有模式，确定性可重放 |
| 境界突破成丹成器布阵 | 复用 StatGenerator 溢出回流 chokepoint（'随机但有上限'同构：火候+药材+运气→品阶/数量带caps），加权抽取+封顶+失败回流 | 同源 Stats.Apply hard-clamp 逻辑 |

## 7. 与 v1.2 戏剧系统接缝

- **因果/业力 → 恩怨债系统**: 因果流'种因→延迟→得果'天然映射 v1.2 恩怨产生/继承/化解多阶段状态机；气运掠夺=零和资源转移作夺嫡/夺宝戏剧资源层。
- **辅助路线产出天然制造关系边**: 供丹=恩义边、夺舍=死仇边，直接喂 Relations。
- **冤孽顶层 + 隐藏血脉多实例**: 觉醒机制（血脉/体质隐藏→线索→揭露→剧变）直接复用；背景预注入隐藏标签，机缘事件揭露。
- **剑奴复仇母题载体**: 被蓄养→反噬→反抗→成长→复仇=6阶段复仇母题天然挂载点。

## 8. 反例守则（明确避免锁死/过度耦合的边界）

1. **不把克制写进路线本体**: 路线只声明 `counter_keys`，克制图独立装配 → 改平衡不动路线。
2. **不把对手 PathId 硬编码进 enum 互引**: 全走 string key + guard 谓词。
3. **不把背景写死进路线**: origin_bonus 单列阈值偏移表，背景与路线 N:N。
4. **不让新维度污染四维**: 隐藏轴/道德/气运按需挂载，四维定和约束神圣不可破。
5. **不在 LLM/决策阶段消费领域 RNG**: VariedSelector/克制掷骰串行消费注入 IRandom（对齐 design §5.6 确定性边界）。
6. **不让 exemplars 回写状态**: 代表作仅 FlavorText 池，绝不进 DomainEvent。

## 克制关系网

# 克制关系网（三条正交轴 + 资源依附层，整数系数·边带 guard 上下文变量·防僵化）

## 轴一·物理 physical（环状制衡，石头剪刀布升维，方向由 guard 翻转）

```
        剑修 ──克(剑锋破横练罩门)──▶ 体修
         ▲                          │
         │克(剑域反锁/已布阵)         │克(近身一拳破/抗法)
        阵修 ◀──克(远程破阵眼)── 法修 ◀──克(贴脸打断读条)── 剑修
         │                          ▲
         └──克(困阵放风筝)──▶ 体修 ──克(远程拉开)──▶ 法修(被)
```

| 攻击方 | 被克方 | 系数 | guard（上下文变量，方向闸） |
|---|---|---|---|
| 剑修 | 体修 | +30% | 无（剑锋破点） |
| 剑修 | 法修 | +30% | 贴脸（distance≤1，打断读条） |
| 体修 | 法修 | +30% | distance≤1（近身糊脸抗法） |
| 法修 | 体修 | +30% | distance>1（远程放风筝消耗）→ 与上行构成距离决定方向 |
| 法修 | 阵修 | +25% | 无（远程精准破阵眼/打断布阵） |
| 阵修 | 剑修/体修 | +40% | arrayed_flag==1（已布好阵；未布阵=最弱战力） |
| 器修 | 剑修 | +25% | tier_gap≥1（法宝品阶压本命剑） |
| 器修 | 法修 | +20% | 无（法宝防御/反制） |

**双向边**: 器修 ⇄ 阵修（造破阵法宝克阵 / 造阵盘阵旗助阵），符号由 `is_hostile` 上下文决定。
**软控双子**: 符修 ⇄ 音修（同抢控场位，效果可叠可冲）。
**雷修/儒修横向破邪全谱**: 纯阳雷力 + 浩然正气天克一切阴邪（见轴二/轴三交叉）。

## 役使系·三态克制三角（活体 vs 死物 vs 寄生）

```
御兽师(活体) ──克(死物针对)──▶ 傀儡师(死物)
傀儡师(死物) ──免疫(无心识/无血肉)──▶ 毒蛊/音修
毒蛊(寄生活体) ──克(夺心控兽/投毒)──▶ 御兽师(活体)
```
- 死物傀儡：免疫毒蛊/音修，但惧雷符（雷克金属/破阵纹）。

## 全域共性·斩首脆弱性（统一'指挥链中断'机制）

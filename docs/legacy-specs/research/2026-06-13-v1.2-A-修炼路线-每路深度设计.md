## 统一升级 schema（数据驱动 CultivationPath）

## v1.2-A 数据驱动 CultivationPath schema（升级版）

### 0. 设计契约（与 v1.0 整数确定性核兼容）
- **不污染四维定和**：四维仍是 `StatBlock`（Force/Internal/Constitution/Insight，Σ=80，cap=30）。所有 per-path 资源（剑意/血气/煞值/气运/itemTier/灵兽栏位…）放在角色侧挂的独立 `CultivationState`，**不进 StatBlock，不进 Σ**。
- **realm 是新维度**：v1.0 `Character` 无 realm。新增 `int RealmIndex`（0..N，每路 N 可不同）到 `CultivationState`，与 `PathId` 一并挂到角色（见 §3）。
- **纯整数 / 禁浮点**：所有公式项、权重、倍率、定标全整数；除法一律整数除（向下取整），右移定标显式写 `>>k` 或 `/d`。
- **可插拔 / 跨表 string key**：`PathId/artId/skillId/resourceKey/counterKey/tag` 全 string key；路线本体只声明自身能力与 `CounterKeys/AttackDimension`，**不硬编码对手**（克制矩阵独立）。
- **声明式**：一条路 = 一份 JSON（`CultivationPathDef`），引擎解释执行；新增/改路不改 C# 代码（除非新增未注册的 `OpKind/ModKind/DerivedProvider`）。

### 1. C# 类型（netstandard2.1，record + init，与现有 `IsExternalInit` 兼容）

```csharp

{

        string PathId, string Name,
        string AttackDimension,              // "physical"|"spirit"|"fate"|"economic"

        PowerFormulaDef Power,               // 声明式整数战力公式
        RealmCurveDef Curve,                 // 独立战力曲线（整数倍率数组+调制）
        IReadOnlyList<ArtCategoryDef> ArtCategories, // 命名功法类目
        IReadOnlyList<CombatSkillDef> CombatSkills,  // 战技池（≥5）

        string CategoryKey, string DisplayName, string Role,

    public sealed record CostDef(

        AddTermWeightStep, AddResourceCap, AddResource, AddFlatDR, SetFlag,
        AddPenInteger, ScalarMul, AddCounterAdj, GrantPassive,

        EffectOpKind Kind, string? TargetKey, int A, int B, string? Note);
```

### 2. 关键升级点 vs 朴素 schema
1. **`PowerFormulaDef` 取代单一硬编码 `Power()`**：每路一棵整数表达式（term-list+modifier-pipeline+realm-curve），见 powerFormulaModel。
2. **`ResourceDef` 列表**：剑意/血气/煞值/气运/Karma/itemTier/soulBond/manaPool/henglian/seaIntegrity/thunderCharge/灵兽栏位统一抽象为「带上下界、可选上限表达式、是否战后清零」的条目，不进四维定和。
3. **`EffectOp` 有限算子集**：功法/战技效果不再自由文本，引擎可解释、可结算、可复现。Flavor 名只进 `Name/Note`，不进数值。
4. **`Curve` 升为 `RealmCurveDef`**（含调制器）：支持双包络（阵修布阵 vs 未布阵）、昼夜倍率、SeaIntegrity 乘子门、itemTier 跟境折扣。
5. **`SelectionRuleDef`**：tier≤realm 闸门、强制 family（风险对冲）、NPC 确定性贪心键，全部数据化。

## 声明式 per-path 整数战力公式模型

## 统一「声明式 per-path 整数战力公式」模型

目标：12 路公式**项不同、权重不同、资源不同、甚至带负权**，却能**同一段解释器**算出整数 `EffectivePower` 并入 `SparAction`。核心：把「公式」从 C# 代码降为**数据**（term-list + modifier-pipeline + realm-curve），由 `PowerEngine` 解释执行。

### 1. 三段式整数管线（12 路共用同一执行框架）
```
EffectivePower = Curve( Modifiers( BaseSum( terms ) ) )
```
- **BaseSum**：`Σ weight_i × source_i`（整数加权和，权重**可负**，如鬼修 Constitution 权 -1）。
- **Modifiers**：有序整数调制流水线（相克盘 ±clamp、布阵 flag 门、昼夜 ×、SeaIntegrity 乘子、续航惩罚 ×3/4、itemTier 跟境折扣…）。**顺序在数据里声明，确定性。**
- **Curve**：`× realmMultipliers[realm] / div`（整数除/右移定标），再叠 `postMul`（开光 ×11/10、夜战 ×3 等）。

### 2. `source` 取值器（让异构项同框）
解释器按 `src` 前缀从统一 `IPowerContext` 取整数：
| 前缀 | 含义 | 例 |
|---|---|---|
| `stat:Force/Internal/Constitution/Insight` | 四维 | 剑修 `stat:Force×4` |
| `realm` | 境界序号 | 体修 `realm×8` |
| `sumArtPower` | 所选功法 tier→power 之和 | 法修 `×1` |
| `res:<key>` | per-path 资源 | `res:swordWill×3`、`res:manaPool×3` |
| `derived:<key>` | 引擎算派生量 | `derived:spellBreadth`、`derived:itemTier`、`derived:henglian`、`derived:rosterWeighted`（驭兽 Σ beastPower×bond/100） |
| `const:<n>` | 常数下限 | 防除零 |

```csharp
public interface IPowerContext {
    int Stat(StatKind k); int Realm { get; }
    int SumArtPower(); int Resource(string key); int Flag(string key);
    int Derived(string key); }   // per-path 派生计算器，纯整数确定性
```
新路只要项能映射到这些 `src` 即无需新代码；仅当需要全新派生量（如阵修 `(2+terrain)`）时注册一个 `IDerivedProvider`（纯整数、可单测）。

### 3. `weightStep`：让「开光/结丹/化形」改权重而不改公式结构
term 可带 `weightStepKey`，实际权重 = `baseWeight + Flags[weightStepKey]`。剑修开光→`swordWillStep=1`→swordWill 权重 4；佛修/器修同理。避免为「升阶改权重」写分支。

### 4. Modifier 算子（声明式、顺序敏感、全整数）
```csharp
public enum ModKind {
    GateByFlag,        // res 子和 × flag（阵修：阵图power × arrayed_flag）
    TerrainMul,        // × (2+terrain)（阵修地利）
    DayNightMul,       // 煞值项 ×1(昼)/×3(夜)（鬼修）
    SeaIntegrityMul,   // × SeaIntegrity/100（魂修识海乘子门）
    SustainPenalty,    // manaPool/cost==0 → ×3/4（法修续航）
    CounterWheelAdj,   // ± counterAdj，clamp ±P0/4（法修相克盘）
    ItemTierLag,       // itemTier < floor(realm*1.2)+1 → 打折（器修跟境）
    QixueHalveHenglian }// 血气=0 → henglian 砍半（体修）
public sealed record PowerModifier(ModKind Kind, string Arg, int A, int B);
```
每个 Modifier 输入上一步整数、输出整数；clamp/除法显式。**同种子+同选择 → 逐字节一致**（满足 v1.0 `DeterminismTests`）。

### 5. 入 SparAction：双值结算 + 攻击维 + 克制矩阵
把现状 `SparAction.Power(c)=Force*2+Internal+Constitution` 替换为：
```csharp
int pa = PowerEngine.Eval(pathA, ctxA);   // ctx 含 stats/realm/资源/flags
int pb = PowerEngine.Eval(pathB, ctxB);
// 克制矩阵独立：按 AttackDimension×CounterKeys×对手 tags 喂入整数 adj
int adjA = CounterMatrix.Resolve(pathA.AttackDimension, pathA.CounterKeys, ctxB.Tags);
int adjB = CounterMatrix.Resolve(pathB.AttackDimension, pathB.CounterKeys, ctxA.Tags);
int effA = pa + Clamp(adjA, -pa/4, pa/4);   // ±P0/4 硬闸，防碾压
int effB = pb + Clamp(adjB, -pb/4, pb/4);
// 绕防（魂修绕物防/命修气运正交/器修落宝）由 CounterMatrix 表达为 adj，仍受 ±P0/4 约束
var winner = effA >= effB ? a : target;
int margin = Math.Abs(effA - effB);
// Relations.Adjust / DuelResolved 维持 v1.0 事件流；
// 丹修/佛修造边、渡化收束、夺器走 EffectOp(AdjustRelationEdge/ApplyStatDelta)，
// 在战技 OnUse 阶段复用 w.AdjustRelation / w.ApplyStat 既有 chokepoint。
```
**关键不变量**：`SparAction` 只认 `int EffectivePower + AttackDimension + counterAdj`，对 12 路一视同仁；路线差异全在 `PathDef` 数据 + 少量已注册 `DerivedProvider/ModKind`。新增路 = 加 JSON，不改 `SparAction`。
**战技伤害**（剑二十三 `Force*3+swordWill*4`、燃血、夺运 `NetFortune/3`、万宝 `Σ itemTier*8`…）走独立 `DamageResolver`，输入同一 `IPowerContext` 输出整数 delta，扣资源经 `ApplyResource` chokepoint。战技只在 `OnUse` 结算，不进基础 `EffectivePower`（与各路材料「战技只进文本/按需结算」一致）。

## 跨路战力平衡

## 跨 12 路战力当量与平衡（防某路碾压）

### 1. 统一锚点：P0 当量带（同 realm 同四维下「裸战力」可比）
- 参考体：四维各 20（Σ=80）、`sumArtPower≈90`、资源半满。把每路 `BaseSum`（未乘 curve）调到**同 realm 下 P0 落同一带**（建议 `P0Band≈[90,130]`，放 `Tuning.P0Band`）。差异由**曲线形状+条件门**体现，而非裸值碾压。
- 校验：每路在 realm=0/4/8、四维=20 计算 `BaseSum`，断言落带内；偏出则调权重不调结构。这是防碾压第一闸。

### 2. 曲线天花板可比（对齐节奏而非绝对值）
| 路 | 末倍率/10 | 形状 | 容错锚（退化态） |
|---|---|---|---|
| 剑修 | 27.0 | 高爆发 | swordWill空/根骨低即崩 |
| 体修 | 7.3 | 稳健耐久 | blood=0 henglian砍半、怕放风筝 |
| 法修 | 4.4 | 均衡续航 | manaPool耗尽 ×3/4 |
| 阵修 | 2.1(布阵×地利) | 双包络 | 未布阵≈裸悟性、偷家必败 |
| 器修 | 15.0 | 厚积晚发 | 脱宝武力=1 |
| 魂修 | 4.0 | 高爆发低容错 | 击穿失败反震识海 |
| 命修 | 30.0 | 厚积+信息枢纽 | 净气运打穿即雪崩 |
| 丹修 | 0.5(directPower) | 资源枢纽 | 直面被秒、需护道人 |
| 鬼修 | 5.5(夜×3) | 昼弱夜强 | 白昼腰斩、噬主反噬 |
| 佛修 | 22.5 | 厚积晚发凸尾 | 前期最慢、破戒清愿 |
| 雷修 | 34.0 | 高伤高自险 | 根骨<thr 触雷噬致残 |
| 驭兽 | 7.0 | 群战协同 | 斩首断线全崩 |

**当量原则**：末倍率绝对值**不可直接比**（BaseSum 权重不同、是否乘条件门不同）。可比的是「理想态 vs 退化态有效战力差」。要求每路**退化态有效战力 ≤ 同 realm 参考体 0.6×**——「高天花板必配真地板」，无白嫖碾压。

### 3. 克制叠加上限：±P0/4 硬 clamp（最关键反碾压闸）
- 单条克制 adj 已 clamp 到 ±目标战力 1/4（法修相克盘、命修 Tempo 削敌沿用）。
- **多重克制不可线性叠到归零**：`effA = pa + Clamp(Σadj, -pa/4, +pa/4)`——即便鬼修同吃佛光+雷劫+纯阳，总修正封顶 25%。防「克制链碾压」。
- 绕防（魂修绕物防、命修气运正交、器修落宝）经 CounterMatrix 表达为 adj，同进 ±P0/4 闸；绕防≠无视战力，只把对手某防御项在 adj 内按 0 计，仍受 25% 上限。

### 4. 资源回报递减 + 负债对冲（防资源流无限滚雪球）
- 命修：净气运=Fortune−Karma；夺「气运高于己者」按差值反弹入自身 Karma/折寿（Backlash）→ 强夺命大者自险=内生天花板。
- 鬼修：噬主度跨阈反噬；选取规则强制 1 个 brake family（镇魂/献祭）——禁纯堆输出。
- 器修/驭兽：itemTier 被 `floor(realm*1.2)+1` 硬封顶、灵兽 power×bond/100 受 bond 削减——外部资源不能越境无限堆。
- 雷修：自险项结算前比 `根骨≥thr=6+realm*3`，不达线整数雷噬+武力衰减——高伤强制配自损。
- **统一纪律**：凡「资源直接进战力」的 term，必须在同一 `PathDef` 配一条**对称负反馈**（耗尽惩罚/反噬/封顶/折扣），由 Modifier 或 EffectOp 强制执行。

### 5. 时间/环境窗口当量（防准备型路无条件碾压）
- 阵修/丹修/器修/命修/驭兽=厚积/枢纽型：低 realm 段曲线刻意 ≤ 剑/体/雷同阶。校验：realm≤3 时这些路理想态 ≤ 高爆发路同阶 1.0×、退化态 ≤0.5×——「前期可被一波带走」是真的，给爆发路开窗。
- 偷家/斩首/焚石断源/拉离灵脉/放风筝 必须在 CounterMatrix 有对应负 adj 或直接置 `arrayed/bond/blood=0`，否则准备型路失衡。

### 6. NPC 生成不破坏定和
选功法只加 `sumArtPower` 与资源，**绝不改四维 Σ=80**（StatGenerator 不动）。贪心选取（`greedyKey`）确定性按四维排序，不引入新随机源——对齐 v1.0 `DeterminismTests/StatGeneratorTests` 的定和+封顶不变量。

### 7. 建议平衡测试 gate（交 auditor）
1. **P0 带测试**：每路 realm0/4/8、四维=20 → BaseSum∈P0Band。
2. **退化态地板测试**：每路退化态有效战力 ≤ 0.6× 参考体同 realm。
3. **克制 clamp 测试**：三重克制下 effA ≥ 0.75×pa。
4. **确定性测试**：同种子+同选择，`PowerEngine.Eval` 逐字节一致（含所有整数除/clamp）。
5. **资源负反馈存在性测试**：每路若有「资源进战力」term，必存在配套负反馈算子（静态扫描 PathDef）。

### 剑修·剑仙  `sword_immortal`  [ · ]
- **特色机制**：【斩道见血·开剑意】剑修不堆肉身,而靠"斩道"开"剑意"。独有机制三件套,均整数、确定性、与他路不同:① 本命剑(BenmingJian):入门祭剑绑定一柄本命剑,记 剑成度jianCheng∈[0,100];斩道实战胜利+5、败而见血+2、闭关磨剑+1;满100则本命剑"开光",powerMetric 中 swordWill 项再+1阶并解锁顶阶剑意。② 剑意值(SwordWill,本路专属资源,初始0,上限=20+5×realm_index):靠"见血"积累——每场切磋/斩杀按 margin 整除得意 min(10, margin/5),用于驱动战技;剑意空=战技哑火(低容错之源)。③ 斩道(ZhanDao)突破:剑修不靠灵气/丹药堆境界,而靠以剑斩道、见血悟意——每升一 realm 需累计斩道里程zhanDaoMiles ≥ 100×realm(实战见血+margin、问剑名宿+30、剑冢悟剑+20),凑满即可斩破境界壁,绝不靠资源喂养(区别于法修买符、体修吞丹)。整套构成"高爆发(剑意满载则单击穿透碾压同阶)、低容错(剑意空/根骨低则一击崩盘)"的真机制而非换皮。
- **战力衡量**：剑修战力=武力为骨、剑意为锋、悟性化意、境界开锋。与默认 F×2+I+C 公式根本不同:本路把武力权重抬到全路最高(单点穿透速度第一)、把悟性经'剑意'转成爆发主项、刻意压低根骨权重(脆皮低容错)、并叠加两项他路没有的资源——本命剑剑成度 与 剑意值swordWill。所有项整数加权,结算=Σ(weight×source)再乘 realmMultipliers[realm]/10(整数除,确定性)。
  - terms: 武力×4(剑修核心:单点穿透/出剑速度,全路线最高武力权重(默认仅×2)) ; 悟性×3(剑意爆发之根:悟性高则'遇强则强'、破绽捕捉快,化为高伤) ; 内力×2(御剑/飞剑/剑气外放的燃料,次于武力(剑修非内力流)) ; 根骨×1(刻意最低权重:剑修脆皮,根骨只兜底不堆肉——低容错的数值来源) ; realm×6(境界序号 0..8,斩道破境的总闸门;再经 realmMultipliers 二次放大成最高天花板) ; 所选功法power×2(本人所选剑法/心法/身法/剑意 各功法 tier→power 之和) ; 剑意值×3(本路专属资源 swordWill:见血积累、驱动战技,满载爆发/清空哑火(高爆发低容错的开关))
- **战力曲线**：高爆发低容错·同境界天花板最高(凸加速曲线:低阶与他路持平甚至偏脆,高阶斩道开剑意后指数拉开;但任一阶剑意空/根骨低即一击崩盘) | realmMul=[10, 14, 20, 30, 46, 70, 108, 170, 270] — realm 0..8 统一映射[三流→二流→一流→绝顶→宗师→大宗师/手中无剑]↔[炼气→筑基→金丹→元婴→化神→炼虚→合体→大乘→剑仙];结算时×倍率/10。逐阶比值 1.4→1.43→1.5→1.53→1.52→1.54→1.57→1.59 递增=凸加速,体现'厚积薄发于斩道、高阶天花板冠绝诸路';但低阶0~2倍率不占优、叠加根骨权重最低,故容错最低。

### 炼体·横世武夫（体修）  `ti_xiu_hengshi`  [外功横练 × 仙侠炼体；血肉之躯铸不灭金身 · physical]
- **特色机制**：本路两条独有真机制，非换名：①【横练罩门 / 减伤层】每名体修维护一个整数"横练值 henglian"=∑所选横练功 tier 效果 + 根骨/3 + realm加成；受击时按"实伤=来袭武力威 − 横练值，下限1"结算——即攻防全面碾压、把绝大多数招式扛成擦伤（区别于他路按比例减伤或闪避）。但每名体修有一处具名"罩门 weakpoint"（如 章门/气海/眉心），针对罩门的攻击无视横练值且×2，构成可被克制矩阵挂钩的硬开关。②【血气池 / 不灭金身】每名体修维护整数"血气 qixue"(0..cap=200)，靠血气吐纳与战技回充；血气可主动燃烧：每点抵消2点已穿透实伤(燃血护体)，realm≥6时每场战斗首次濒死自动燃尽血气复活一次(不灭金身)。这把体修锁成"稳健耐久、攻防碾压"，且把唯一弱点钉死为"被持续消耗/放风筝拖干血气与体力"——血气=0时横练值砍半、不能复活。资源不进四维、独立字段，不污染定和。
- **战力衡量**：体修战力=根骨主导的肉身堆叠，与他路公式互不相同。剑修=武力×2+剑意爆发(高方差)；法修=内力+法术库广度；魂修=魂力绕物防——而本路以根骨(权3)为首、武力(权2)次之，叠 realm 厚势(权8/阶)、所选横练功 power、血气资源折算；无悟性项、无爆发项，故体修同四维下战力更平更高底更耐久，赢在底盘与续航而非尖峰。各项整数加权求和，结果整数；血气见底时资源项归0且横练值砍半，量化'惧消耗与放风筝'。
  - terms: 根骨×3(Constitution：本路最重权，肉身堆叠主轴(≤cap30→最高90)) ; 武力×2(Force：横练拳脚输出与破横练，第二权重) ; 内力×1(Internal：仅供血气吐纳转化效率，弱权) ; realm×8(realm_index 0..9：每境界+8厚势基线，稳健底盘而非尖峰) ; 所选功法power×1(选定横练功 power(=tier×6)直接入战力，横练越厚战力越高) ; 血气×1(专属资源 qixue：按 qixue/20 折算(满池约+10)，见底则归0且 henglian 砍半)
- **战力曲线**：稳健耐久：低开高走、单调平滑、无尖峰。早期(realm0-2)倍率最低，是四路里最弱的开局(炼体起手要拿肉身硬挨)；中期(realm3-6)随横练成型稳步压过剑修同阶；后期(realm7-9)金身大成，攻防双高且容错极大，但封顶倍率低于剑修爆发顶——以'扛得住、磨得死'取胜而非一击毙杀。曲线差分恒为正且趋缓，体现厚积稳进。 | realmMul=[10, 13, 17, 22, 28, 35, 43, 52, 62, 73] — 整数倍率，realm0..9。相邻差 3/4/5/6/7/8/9/10/11 单调递增但加速度温和——对照剑修'高爆发低容错'应给前期低、后期某阶跳变大的数组，本路刻意无跳变。战力曲线只乘在最终整数战力上：finalPower = ((根骨×3+武力×2+内力×1)+realm×8+横练功power+血气/20) × realmMultipliers[realm] / 10（/10 为定点缩放，保持整数）。
- **修炼途径**：修什么：练的是'血肉之躯'本身——以《易筋》洗髓伐毛、横练皮膜筋骨成铁，把肉身当作法器反复淬炼。靠什么资源：①血气(qixue)，由'血气吐纳'功在静修与进食/疗伤时积累，是燃血护体与不灭金身的燃料；②横练值(henglian)，由修成的横练功永久叠加，是常驻减伤底盘；③根骨悟性，决定淬炼上限与突破成功率。如何突破境界(realm_index 0..9，武侠↔仙侠同台)：realm0-2'锻皮淬肉(三流→二流)'靠日课锤炼涨根骨与横练值；realm3-4'横练罩门→淬骨(一流→小宗师，对炼气→筑基)'须以重击反复轰击罩门将其'淬实'(罩门威胁随之下降)；realm5-6'血肉开窍/伐毛洗髓(宗师→大宗师，对金丹)'须血气满池冲关，成则开启不灭金身；realm7-9'铜皮铁骨→不灭金身/陆地神仙(手中无招，对元婴)'须根骨≥阈值且血气满，把肉身炼成准帝级金身。突破判定吃悟性但战力公式不吃悟性——这是本路独有的'笨功夫'设定：会练的未必聪明。突破失败损血气不掉境界(耐久路线低容错惩罚轻)。

  - 嗜血催狂 [t2]：催动指定灵兽进入狂化：该兽 beastPower×150/100 持续3 tick，结束后 bond −20(透支纽带换爆发)（内力中、bond 透支）
  - 断线应急·影替 [t4]：本体将受致命/斩首打击时触发：指定一只'护主兽'代受并把本体瞬移出阵，阻止'本体斩首→全兽崩散'一次(克脆性的保命技)；与'舍身护契大法'联动可彻底化解一次斩首断线（内力高、护主兽当场脱契损失）
  - 万兽齐鸣·镇魂 [t3]：全在役灵兽齐吼，对范围内敌方'音修乱兽/精神扰动'反制并清除我方被乱状态，bond 全体回 +10(反·乱兽)（内力高）
  - 灵兽献祭·爆体 [t4]：献祭一只契约灵兽，造成等于其 beastPower×300/100 的一次性范围爆发，该灵兽永久移出栏位(资源换斩杀)；其余兽 bond 因纽带网震荡 −5（永久损失一只灵兽、bond 网震荡）
  - 召兽归阵 [t1]：将散落/被乱/逃逸的灵兽强制召回阵中并重置其阵位，恢复兽阵倍率触发条件(被打散后重整旗鼓)（内力低）
- **选取规则**：每个驭兽师角色从本路按"1主修类目功法 + 1契约类功法 + 1兽阵类功法 + 2~3战技"装配，且独立持有一份'灵兽栏位'(初始1只低阶契约兽，随 realm 增长收服)：具体=从【驭兽术】选1(决定压服/收服能力与可驭品阶)，从【契约·御灵】选1(决定 bond 上限/纽带韧性/抗夺抗乱)，从【兽阵·役使】选1(决定群战协同倍率与阵眼数)，从【灵兽心得·培育】按已收服兽数被动挂载(每只兽可挂1条培育心得，影响该兽成长曲线)，再从战技池选2~3个主动招式(至少含1个'群体协同/召唤'战技与1个'断线应急/护主'战技，以匹配'怕斩首'的脆性)。装配后该角色战力 = 按 powerMetric 对其本体四维 + 当前灵兽名册(每兽 power×bond) × 兽阵倍率求和，全整数、可插拔；增删一只灵兽或一条功法即重算，不污染四维定和(灵兽 power 是独立资源条目，不计入武力/内力/根骨/悟性之和)。

## 缺口（gaps）

- realm 维度在 v1.0 Character 上不存在：需先落地 CultivationState（PathId+RealmIndex+Resources+Flags+Roster）并定义突破/晋阶 action，否则曲线 realmMultipliers 无索引源。这是 schema 落地的前置 gate。
- 克制矩阵 CounterMatrix 是 attackDimension/counterKeys/绕防/±P0/4 clamp 的承载体，但 12 路材料只声明各自 counterKeys 字符串，未给统一矩阵表与 tag 词表。需独立设计 CounterMatrix（含 spirit/fate/economic 绕防语义、element 相克环、anti_evil/纯阳灭阴倍率注入），否则 SparAction 的 adj 无法计算。
- 各路 realmMultipliers 长度不一（剑修9/体修10/法修9/阵修8/魂修7/命修10/丹修8/鬼修8/佛修10/雷修9/驭兽9），realm_index 上限 N 每路不同；需统一一张 realm↔武侠/仙侠境界映射表并规定越界（选 tier>realm 上限、索引越界）的钳制/报错策略。
- P0 当量带的具体上下界（建议[90,130]）与各路权重的精确标定未做：材料给了权重但未跨路对齐 BaseSum；需一轮数值标定+P0带测试才能断言无碾压。
- 丹修双战力（directPower 受击 vs alchemyLeverage 威胁度）需 SparAction 明确：受击用哪个、是否引入威胁度独立评估通道；当前 SparAction 只有单一 Power 比较。
- 祭炼/craftScore、布阵 setup_progress、夺舍成功率、炸炉等随机结算需复用 IRandom.Split(streamId) 子流约定，确定性子流 id 分配规则（per-character/per-action）未定义，否则多随机源会破坏逐字节复现。
- 资源相互作用的跨路边界（命修夺运作用他路 Fortune、器修落宝夺他路 itemTier、丹修毒丹改他路四维、驭兽毒蛊夺兽）需一个跨路资源/状态访问协议；当前 CultivationState 是 per-path 私有，跨路写入需经独立 InteractionResolver 与权限闸（不污染四维定和、经 ApplyResource/ApplyStat chokepoint）。
- 战技 OnUse 的 DamageResolver 算子集（Force*3+swordWill*4、Σ鬼兵power*2、NetFortune/3、itemTier*sum*8…）在 EffectOp 枚举里用 note 占位，未枚举为结构化 op；若要完全数据驱动战技伤害，需把伤害表达式也降为可解释整数表达式 AST，否则部分伤害仍需 per-path C# 分支。
- NPC 确定性贪心选取（greedyKey）只给了剑修/驭兽等少数原型映射，12 路与 ArchetypeKind(Martial/Knight) 的完整映射表未定义。

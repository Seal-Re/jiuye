## 1. 三轴解耦（收敛后的唯一编码模型）

根本张力：12 路 `realmMultipliers` 长度不一（剑修9/体修10/魂修7/命修10…），且同 RealmIndex 在不同路语义不同（剑修 realm3=绝顶/元婴当量；体修 realm3=一流/筑基当量）。强行统一会毁已标定曲线；裸绑 RealmIndex 会让跨路当量荒谬。

**收敛解 = 三个解耦整数轴**：

| 轴 | 取值 | 存储 | 权威用途 |
|---|---|---|---|
| **flatIndex**（权威） | `0..Nₚ-1`，Nₚ=该路 `RealmCurveDef.MajorMul.Length`（即旧 realmMultipliers，7~10，**不改数组**） | **CultivationState.RealmIndex 单字段**（复用 A.0 既有字段，零新增可变字段） | 战力 curve 索引 / tier 闸 / 突破阈值 / Clone 续跑 |
| **(MajorTier, SubLevel)**（双层语义视图） | 由 flatIndex 经 per-path 前缀和投影**纯函数派生，不存字段** | 无（读时算） | 显示 / 突破节奏 / 劫触发 / **小境界进度** |
| **UnifiedTier**（跨域刻度） | `0..12`，由 per-path 表把 flatIndex→UnifiedTier | 无（读时查 per-path 表） | 寿元档 / 天劫档 / 跨域同台 / 碾压闸 |

**裁决 1（编码）**：采 A2 前缀和投影，**否决** A.0 固定基数 `major*9+minor`（A2 自审 C1 已证它对不等长小境界产空洞编码）与 C 的独立可变 `MinorLayer` 字段（双源不一致 + 续跑多一拷贝面）。

**裁决 2（小境界是否进战力）**：A.0 采**小境界不进 PowerEngine 主战力**（A.2 保守方案），PowerEngine 终段公式 `final = raw × MajorMul[flatIndex] / 10` 真正一行不改、A.0 曲线逐字节不变、INV-CURVE 风险归零。小境界只用于**显示 / 突破节奏 / 寿元 / 劫**。小境界进战力（minorScale 定点乘子）**整体推 A.1**，且届时必须 per-path 由该路 MajorMul 最小相邻比反推上限（否决 A.0 的全局 2.3× MinorMul——实测它让体修小境界满层≈2 个大境界，破 INV-CURVE）。

**裁决 3（双层细分粒度）**：小境界数 **per-major 数据驱动**（`MajorRealmDef.SubLevelCount`，每大境界自声明，允许炼气9、化神3、大乘1），**否决** A.0 全路 MINOR_STEP=9 与 C 的全局 9/5 两套（遮天轮海4/道宫5/化龙9 实证小境界数本就 per-major 不规则）。当 SubLevelCount=1 时该大境界无小境界（不产空壳 SubLevel）。

**裁决 4（跨域当量绑哪个轴）**：寿元/天劫**绑 UnifiedTier，不绑裸 RealmIndex**（A.2 的"绑 index"被 §1 同 RealmIndex 跨路语义不一致证伪）。

### 1.1 编码（纯整数可逆，前缀和）
```
RealmLayout.SubLevelOffsets = SubLevelCount 前缀和（长度 M+1, 末元素 == Nₚ == MajorMul.Length）
encode: major = max j 使 offsets[j] <= flatIndex ; sub = flatIndex - offsets[major]
decode: flatIndex = offsets[major] + sub
INV-REALM-1: decode(encode(f)) == f 且 0 <= sub < SubLevelCount[major]
```
- 投影纯函数、不存字段 → Clone 只拷一个 int（R-NF2 命门，零新遗漏面）。
- 与 SubLevelCount=1 兼容：该 major 段长 1，sub 恒 0，不产空壳。

### 1.2 UnifiedTier 跨域投影（替代裸 RealmIndex 绑定）
```
RealmCurveDef.UnifiedTierOf[major]  // per-path int[], 长度=M, 每元素∈[0,12]
charUnifiedTier = UnifiedTierOf[ MajorTier(flatIndex) ]
```
- **武侠↔仙侠当量裁决（修正"陆地神仙=化神"错锚）**：联网核验雪中悍刀行/武评——陆地神仙（天人境）是武道**绝对顶点**，不应锚到仙侠第五境化神（上还有炼虚/合体/大乘/渡劫/飞升5层）。采**方案 A（同台可战）**：武侠"境界数少但每境当量高"，陆地神仙/手中无招锚到 **UnifiedTier 10~11**（大乘/渡劫量级）；武侠绝顶锚 UT7~8（化神~炼虚），非 UT3。武侠路 MaxMajor 小但 UnifiedTierOf 跳档大（招式境界压缩、单境跨度大）。
- **缺档处理**：魂修仅 7 major，其 UnifiedTierOf 必跳档（如 [0,2,4,6,8,10,12]）。同台可战的判据是"同 UnifiedTier 战力当量"，**不要求每路在每个 UT 都有锚**。跨域 spar 时若两方 UnifiedTier 相等即同台；某路无该 UT 锚不影响（它根本不会停在那个 UT）。

## 2. 战力放大（全整数，接入 A.0 PowerEngine，小境界不进主战力）

- A.0 基线**不改**：`final = raw × MajorMul[flatIndex] / 10`（整数除）。`src:realm` 项解析为 **MajorTier**（=投影 major，不是 flatIndex），off 时 flatIndex=0→major=0→旧值，38 测试逐字节不变。
- **大碾压闸（新增，前置，单向）**：`if atk.UnifiedTier - def.UnifiedTier >= CRUSH_TIER_GAP: defEff = defEff × crushNum / crushDen`。
  - **修正鬼谷差评（确定性碾压无变数）**：碾压幅度**随差距渐进、非一刀切**，且**留越级反杀窗口**：
    - `gap==2 → ×3/4`；`gap==3 → ×1/2`；`gap>=4 → ×1/4`（CRUSH_TIER_GAP 默认 2 起步、渐进）。
    - 碾压**只削守方基数**，不加攻方（防双向爆炸），且置于克制 adj（±P0/4）**之前**，与路内越级机制各自独立 clamp 不互乘（防归零）。
    - 越级反杀通过既有"克制链 ±P0/4 + 消耗品 + 地形 + 有界 roll"叠加，可在 gap<=2 时翻盘（设计意图：碾压是大概率不是确定）。
  - **碎步回滚**：`UnifiedTier 差 < CRUSH_TIER_GAP` 时碾压判定 no-op，结果 == 纯 A.0 逐字节（INV-POWER-3）。
- **小境界（A.0）不进 PowerEngine**：SubLevel 仅影响显示/突破节奏/寿元/劫。INV-CURVE（小境界圆满 < 下大境界初期）在 A.0 **天然成立**（小境界根本不放大战力）。

## 5. 五类失败模式（全整数触发器 + 恢复路径）

| 模式 | 触发器 | 终态 | 恢复 |
|---|---|---|---|
| 走火入魔 | innerDemon≥80（致死≥95） | 危险态/可救 | PurgeRoll(随 innerDemon 放宽)+外力+疗伤 |
| 卡瓶颈 | bottleneckStreak≥6 | 滞留/可解 | 天材+顿悟+外力 |

## 10. 整数确定性落地

### 10.1 全整数禁浮点
所有阈值/gain/倍率/TribScore 均 int/long；定点显式 `×num/den`；概率门用整数比较+有界 RNG（`Roll=rng.NextInclusive(lo,hi); 成功 ⇔ score+Roll≥gate`），无 float 概率。IL 浮点扫描**新建**覆盖 `Jianghu.Cultivation`。

### 10.2 不破四维（**诚实口径，修正伪不变量**）
> **修正"运行期 Σ=80 神圣不变量"（critic blocker）**：v1.0 `TrainAction` 涨单维把 Σ 从 80 抬到 81+（`ActionsTests:34` 实证 Force>20），运行期 Σ **从来不是 80**。Σ=80 仅是**生成期契约**（`StatGeneratorTests` 只在 generate 断言 sum==80）。
- 诚实表述：四维定和 = **生成期契约**；运行期 Apply 只保 per-维 [0,cap]。
- 修炼不涨四维的理由是"**realm 与四维解耦**"（per-path 资源进 CultivationState 独立轴，符仙侠惯例），**不是"守 Σ"**。
- 所有 per-path 资源/phase/progress/innerDemon/foundation/lifespanBonus/神识 **进 CultivationState 不进 StatBlock**。
- 走火"伤根基" **不扣维值**（避免误导性"破 Σ"叙事），改施 `DeviationDebuff` flag，PowerEngine 读 flag 施战力惩罚（可恢复，用 EffectOp 战力修饰算子——**不是改四维**）。
- **唯一减四维路径（雷噬永久衰减 Force）**：必须新增 `StatBlock.ApplyDecay(k, magnitude>0)` 只减接口，或在 `EffectOp.ApplyStatDelta` 调用点加 `assert delta<=0`——**修正"Apply 正 delta 无差别放行、四维污染入口敞开"（critic）**：当前 `StatBlock.Apply` 对正 delta 同样接受，靠调用方自律不够，需机器强制。

### 10.3 神识/资源是独立字段非四维派生（修正双源 critic）
> **修正 A.2 `DivineSense=Insight*2+RealmIndex*5` 既派生又存储的双源矛盾**：所有 cult 资源一律**独立存储字段**（进 Resources Dict），初值可由公式播种（`Initial=Insight*2`）但此后独立演化，**绝不定义为四维恒等派生**（杜绝改 Insight 隐式改神识的耦合污染）。

### 10.4 RNG 确定性边界（修正流号冲突）
> **修正"Split(5)=drama 已占"假设（critic）**：实读 WorldFactory 仅 Split 1..4，**无 Split(5)**。戏剧亦未实现。
- **修炼占 Split(5)**（当前第一个空闲号，append-only），戏剧顺延 Split(6)。**否决 A.2 复用 `_domainRng`（=Split2 v1.1 世界事件 reserved，混用污染两特性）**。
- `cultRng` **只在 cultivation-on 分支构造，off 一次不 new**（沿用 reserved 未消费不改流编号纪律），补 off 测试断言 root 及各子流 GetState() 逐字节不变。
- 劫 roll 子流 id 统一含 `(charId, flatIndex, cultPhase, attemptCounter)` 全四元（**修正 A.2 hash 缺 phase 致同 tick 多劫复用结果**）。
- 同 (种子,配置) 全流程逐字节复现；Clone（拷 RealmIndex + Resources/Flags Dict）续跑一致。

### 10.5 long 防溢出
`final=(long)raw×MajorMul[flatIndex]/10`，clamp POWER_CAP=2_000_000_000。

## 体量裁决（bodySizeVerdict）
体量够大（裁决：PASS，但纠正『级数=体量』的错误度量）。依据：(1)纵向刻度——双层后单路 Nₚ(7~10)大境界 × per-major 小境界(炼气9/化神3/大乘1 不规则) ≈ 数十级，叠 10 态状态机 + 三劫数据驱动 + 五类失败模式 + 破障四法，纵向纵深足够支撑长跑。(2)横向——12 路各走不同劫型/突破资源/UnifiedTier 投影/失败特色，且要求可证伪差异(同 UT 胜率[40,60]%+最优策略两两不同)，非换皮。(3)关键纠偏(对标太吾冲穴/鬼谷无限刷差评)：体量裁决指标从『境界级数』改为『平均每 N 次修炼出现一次有后果决策分叉』——纯 progress 累加(MinorAccumulate 常态)是单调重复，真体量来自决策分叉(日课练快/练稳微决策 + 破障四法 tradeoff + 劫型应对 + 失败恢复)。当前设计常态决策分叉频率偏低(大境界突破才触发高价值分叉)，已加日课微决策补常态，但高频奇遇/资源事件多推 A.1——故体量『结构上够大』成立，『可玩性单调性』的彻底解决部分依赖 A.1，本稿如实标注于 remainingOpen，不夸大为已解决。

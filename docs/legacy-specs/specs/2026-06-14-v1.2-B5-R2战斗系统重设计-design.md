## 0. 范围 + 与现状关系

**重设计目标**：把 A.0 的「单击 power 比大小」战斗，升级为**三层短回合对拼**——① 通用战斗维 ② 通用武技池 ③ 门派专属战斗方式 + 相克——简单文字式、确定整数、多 NPC 可跑。**平衡(INV-CROSS)在 R2 之上标定**（pe 设量级，招式/相克给质感不破同 UT 平价）。

**不做**：实时/帧战斗、复杂连招树、装备系统。**off 不动**：cultivation-off 走 legacy `Power=Force×2+Internal+Constitution`（`SparAction.cs:33`），R2 只替 **ON 分支**（`EffectivePower` 的 Cultivation 路径）→ **off 逐字节天然守**（off 无 Cultivation 不入 R2）。

## 1. 三层结构（用户定向 → 落地）

| 层 | 内容 | 数据来源 |
|---|---|---|
| **通用战斗维** | 外功/内功/护体招架/身法闪避（所有角色，含散修） | 现有 4 stats 映射（§2） |
| **通用武技池** | 公共招式（基础拳脚/吐纳护体/闪身），任路可用（散修亦有基础招） | **新增**小型 `PublicKungfu` 源（§3.3，唯一实质新内容） |
| **门派专属战斗方式** | 各路招牌招式 = 激活其现有 CombatSkills（剑修剑招/魔修心魔噬/丹修弱战） | **现有 CombatSkillDef 激活**（A.0 装载却不参战，本稿填洞） |
| **相克** | 内攻克外防 / 外攻克内防 / 流派软克（无硬克 ±P0/4） | **复用现有 SituationalResolver**（§5） |

> 北大侠客行范式：基本功夫(内功force/轻功dodge/招架parry)=通用维；公共武功(金蛇剑法/七伤拳…)=通用武技池；门派武功=专属。本稿同构。

## 2. 战斗维映射（4 stats → 战斗角色）

| 战斗维 | stat | 战斗作用 |
|---|---|---|
| 外功（attack-外） | Force | 外功招伤害基 |
| 内功（attack-内） | Internal | 内功招伤害基 + 战斗 HP 厚度 |
| 护体招架（defense-parry） | Constitution | 减伤（招架值） + 战斗 HP |
| 身法闪避（dodge/tempo） | Insight | 闪避减伤 + 出手（先手/多发，A.1+ 可选） |

## 3. 招式模型

### 3.1 招式定义（复用 CombatSkillDef，A.0 已有 Id/Name/Tier/OnUse/Cost）
- **威力** = `Tier`（已有，1-5）。
- **内外功属性** = 用所属路的 `AttackDimension`（已有 flavor 字段）派生：剑/刀/拳脚=外功，法/符/雷/内息=内功，控/毒/音=特殊（按 SituationalTags）。**不加 schema**（path 级 profile 够用；per-skill 内外功细分 defer）。
- **OnUse 效果**：A.0 已有算子；R2 起可触发（如加伤/破防）；首版可仅用 Tier 作威力，OnUse 留接口（不复杂优先）。

### 3.2 招式选择（确定，无随机）
- 角色可用招 = `ChosenSkillIds`（门派专属，定路时选）∪ 通用武技池基础招。
- 第 r 回合用招 = `可用招[r mod 可用招数]`（轮转，确定）。无专属招 → 用通用基础招（散修可战）。

### 3.3 通用武技池（唯一实质新内容，小）
- `PublicKungfu`：一组公共招式（如 `基础拳脚`Tier1外功 / `吐纳护体`Tier1内功+减伤 / `闪身`Tier1闪避）。所有角色默认持有（兜底，确保人人能打）。
- 后续可扩学习型公共武功（金蛇剑法类）= 加数据行（L0），不改引擎。本稿只定**池结构 + 3-5 基础招**。

## 4. R2 结算（短回合，确定整数）

```
战斗 HP：  HP = pe                         // pe = PowerEngine.Evaluate（已校准，同 UT 同量级 → 平衡锚）
回合：     固定 N = 3
for r in 0..N-1:
    moveA = selectMove(A, r); moveB = selectMove(B, r)
    dmgA = damage(A→B, moveA); dmgB = damage(B→A, moveB)   // ① 同回合双方先各算伤（读改前 HP）
    HP_B -= dmgA; HP_A -= dmgB                              // ② 同时扣 —— 无先手偏置（关键：早退KO会让初始者抢末击必胜，破[40,60]%）
    if HP_A <= 0 || HP_B <= 0: break                        // 任一归零即止
胜者 = HP_A > HP_B ? A : (HP_B > HP_A ? B : Tiebreak(A,B))   // 余血高者胜；恰平 → 确定 tiebreak(CharacterId 序，罕见同路镜像局)
margin = |HP_A - HP_B|                       // 喂 DuelResolved（关系/叙事）

damage(atk→def, move):
    atkStat = move.内外功==内 ? Internal(atk) : Force(atk)
    raw     = pe(atk)/D0 × (W0+move.Tier)/W0 + atkStat×Ks    // pe 设量级 + 招式威力 + 攻击维
    mit     = (Constitution(def) + Insight(def)) × Kd          // 招架+闪避 减伤
    kè      = SituationalResolver.AdjPct(atk vs def)            // 相克 ∈ ±P0/4（复用 §5）
    dmg     = clamp( (raw - mit) × (100+kè)/100, Floor, PowerCap )
```
- 全整数 long 防溢，每步 clamp，禁浮点（IL 守）。
- 参数 `D0/W0/Ks/Kd/Floor`（量级/威力/攻防权重/保底伤）= **标定旋钮**（§7 INV-CROSS 调之）。
- 无 RNG（确定）；如需软扰动可接现有对拍 rng（确定 split），首版纯确定。

## 5. 相克层（复用 SituationalResolver，零新引擎）

- 现 `SituationalResolver.AdjPct(SitContext)` 已据双方 SituationalTags + AttackDimension 出 ±P0/4 软 adj（零 PathId，已建）。R2 每回合 damage 乘 `(100+kè)/100` 即接入。
- **内外功相克**（研究实证「内功招对外防高伤害大」）：作为 SituationalEdge 数据行加入（如 `轴=内攻 when 对方护体高 → +adj`），无需改引擎，加数据（L0）。
- 流派软克：现有边表延续，环环克制无硬克（红线：无 CounterMatrix）。

## 6. 文字叙事（R2 质感落点）

- 逐回合叙：`「甲」运[招式A]攻，「乙」以[招架/招式B]挡——[内力雄厚破其守/被身法卸去]，伤 N`。
- 末回合：`「甲」三招之内压定「乙」，胜（余 N）` 或 `「甲」一招[招式]KO「乙」`。
- 落地：`DuelResolved` 保留（winner/loser/margin 喂关系）；回合细节经**可选 narrator 解析器**注入 Chronicle（同 A1.5 realmDesc 模式，off 传 null 回退旧「切磋胜」→ off 字节守）。

## 7. 平衡整合（INV-CROSS 在 R2 之上）

- **pe = 量级锚**：HP + damage 量级均 ∝ pe（已校准 §B5 平衡稿：mul=target×10/BaseSum）。同 UT 典型角色 pe 同 → HP/damage 同量级 → R2 趋 50% → C1 [40,60]% 构造性近似成立。
- **INV-CROSS gate（改 R2 版）**：同 UT ≥2 战斗路典型角色（带招式 loadout）跑 R2 N 局 → 胜率 ∈[40,60]%。招式/相克给的偏差须落带内，否则回调 D0/W0/Ks/Kd 或个别路 pe。
- **辅助路自然弱**：丹/阵 战斗方式 = 少/弱招式 + 低 pe → R2 自然败给战斗路（无需手调突兀 mul）。其战斗当量 UT = R2 实测落点（解 A1.4，比纯 power 矩阵更真）。
- **寿元解耦**：辅助路寿元/劫绑「修为 realm 阶」非战斗 UT（丹帝寿长打不过）→ 属阶段4，本稿只定战斗 UT 锚。

## 8. determinism + off（红线）

- R2 **仅 ON 分支**（`EffectivePower` self.Cultivation!=null）。off（散修/cultivation-off）走 legacy `Power`，**逐字节不变**（off 不入 R2，同 A1.5 Chronicle 改法）。
- 全整数定点，无浮点（IL 扫描守）。同种子同输出（无 RNG 或确定 RNG）。
- 改后必跑：全量绿 + `OffByteIdenticalTests` + IL 浮点零 + 新 R2/INV-CROSS 测试。

## 12. 自洽验证（worked example，证"设计通"）

两典型角色 @ UT8（校准后 pe≈1000），中庸 stats 各 20，Tier-3 招，D0=3/W0=10/Ks=1/Kd=1/Floor=10：
- 单回合 raw = 1000/3×(13/10) + 20 = 433+20 = 453；mit = (20+20)×1 = 40；无相克 dmg = 453-40 = 413。
- HP=1000，**同时扣** 413/回合：r0 后双方 587；r1 后 174；r2 双方各算 413 同扣 → 双方 -239 → 均 ≤0 止。**完全均势 → 余血恰平 → Tiebreak（罕见，仅同路镜像）**。
- 真实异路：A 招式 Tier/AttackDimension 与 B 不同 → dmg 微异（如 A 出 Tier4 招 dmg 430 vs B Tier3 413）→ r2 后 A 余 -227 > B 余 -239 → **A 胜（余血高），margin 12**。**微差定胜负，趋 50% 分布** ✓ C1。
- 加相克：A 内攻克 B 外防 +10% → A 每回合伤 ×1.1 → A 累计削 B 更多 → r2 末 B 余血更低 → A 胜。**相克可决但 ±P0/4 软、不碾压**（碾压留 UT 差≥2，C2）✓。
- 辅助路丹 @ 同 realm 但战斗当量低（pe≈低 + 弱招 Tier）→ HP 薄 + 伤低 → 速败 → 战斗 UT 实测低（解 A1.4）✓。
- **结论**：同时结算去先手偏置 + 3 回合 + pe 锚 + 相克软调 → 同 UT 异路微差趋 50%、相克给质感不碾压、辅助路自然弱。**自洽可标定**。

## 14. 迭代决策（本会话达成，暂存待续 — 下次折入 §1-13 主体 + 设计模块系统）

> 用户定向迭代（2026-06-14 会话）。已拍：R2 选定 + 简版(功法门控防御,无真气v1) + 法宝折入 + **模块化效果系统+稀有度**(核心架构升级)。三段决策如下，下次细化并入主体。

### 14.1 功法门控防御（用户修正"闪避"，调研验证）
- **原则**：战斗能做什么 = 修了什么功法定，**stat 只调强弱不自带能力**（仙侠原味；调研：踏雪无痕需内功+轻功技艺、九阳护体反弹靠内功心法、凡人修仙真气护体）。
- **防御三模式**（§2 闪避挂 Insight 当通用 = 错，改为）：
  - 硬抗（人人，floor）= 体质削减伤害。
  - **护体真气**（需修内功心法）→ 减伤。
  - **闪避**（需修轻功/身法功法）→ 概率避（用户点：没轻功不能闪）。
- 公共武技池（§3.3）即轻功(神行)/吐纳护体/基础招 之家——靠 pick 得能力。

### 14.2 法宝配套效果（用户问"法宝如何在战斗体现"）
- 现状诚实：法宝**已设计极深**（器修 itemTier 主导 power/soulBond/双驱/落宝/万宝齐发；魔 demonWeapon；符 stockFirepower；法 御剑），但**只反应到 power 标量**，未在 R2 回合体现效果。
- 设计：**法宝 = 数值底 + 配套功法解锁的战斗效果**（统一用户4选项，功法门控一致）：
  - (a)数值底：itemTier→pe 量级（已有）。无配套功法=裸数值。
  - (d)配套御器功法→解锁：(c)异化招(飞剑→御剑ranged)/(b)法宝护体(护罩)/(b)特殊效果(困阵/落宝夺敌器)。
  - 器修=御宝流（多法宝/万宝齐发AOE/落宝）= 其专门战斗方式。
- 待定（用户选项4）：法宝人人可持 vs 仅器/法/魔等器系路。

### 14.3 模块化效果系统 + 稀有度（核心架构升级，用户要求）
- **要求**：战斗效果**模块式拼接**（多模块效果+数值拼），新功法=集成复用模块；每功法**有机制非纯数值**；机制分**普通/稀有/唯一**稀有度；**越好的功法越复杂**（更多/更稀模块）。
- **锚到现有**：`EffectOp` 算子原语**已存在**（`ArtDef.Effects` / `CombatSkillDef.OnUse` 均为 `IReadOnlyList<EffectOp>`）= 用户的"模块"。现有 EffectOpKind 多为装配期 stat/资源算子（AddPenInteger/AddResource/AddTermWeightStep/Passive/AddResourceCap），**缺战斗期机制模块**。
- **设计方向**（待细化）：
  - 模块 = EffectOp（Kind=机制 + 参数）。功法/招式 = 模块列表拼接。
  - 扩 EffectOpKind 加**战斗期机制模块**：御剑(ranged)/护体(减伤)/闪避(evade)/困(stun)/落宝(steal)/反弹(reflect)/噬(lifesteal+innerDemon)/连击(multi-hit)/破防(ignore护体)/蛊毒(dot)…
  - **稀有度**：模块标 普通/稀有/唯一（加字段或独立 registry）。功法复杂度 = 模块数 + 稀有度档。越好越复杂。
  - R2 回合引擎**应用模块**（每招式 fire 其 EffectOp 模块 → 各模块改这一击的 clash）。
  - 复用性：模块库一处定义，各功法引用（开闭，加功法=组合既有模块 + 偶尔加新模块）。

### 15.1 网文交叉验证（模块族真实性）

## 1. 日课微决策 — 收敛为单一 4 路 DailyMode（B 批判 P3 修：破"伪4选项塌缩"）

### 1.1 唯一子动作集合（作废 A2=2路 / A3=3路+守一 的发散）

裁决：采 **D 的 4 路**（Fast/Steady/Comprehend/Roam），并吸收 A3 守一语义到 Steady（Steady 已 innerDemon−1+Foundation+1，覆盖守一回血意图，不另加第 5 路免组合爆炸）。单一定点乘子表（整数 ×num/den，兜底增量≥1）：

```
DailyMode { Fast=0, Steady=1, Comprehend=2, Roam=3 }
Fast:     progress×6/4(1.5×), innerDemon+2
Steady:   progress×3/4(0.75×), innerDemon-1(下限0), Foundation+1
Comprehend: progress×1/2(0.5×), EpiphanyRoll<Insight-18→breakProgress+25 或 comprehension+ 或 daoHeart+5(多结局 cultRng.Split 抽)
Roam:     progress×1/4(0.25×), innerDemon-2(下限0), Move相邻节点, 奇遇暴露×3(§5 cap内)
```

### 1.2 破"NPC 自动驾驶塌缩成确定锯齿"（B 批判 P3 核心修）

批判员证实：纯整数贪心给定 (innerDemon,Phase,Insight,arch) 唯一确定 mode → 长程退化为可预测锯齿循环（鬼谷"最优循环"差评本质）。**修复 = 引入滞回(hysteresis) + 把成功判据从"选项数"改为"轨迹熵"**：

```
DailyMode 贪心(actor, cult, arch):   // 纯整数, 无新RNG源, 带滞回
  // 滞回带: 用两个阈值(进入65/退出50)避免在单一阈值上抖动锯齿
  if cult.innerDemon >= DEMON_DANGER_ENTER(65): _demonGuard=1
  if cult.innerDemon <= DEMON_DANGER_EXIT(50):  _demonGuard=0   // 滞回状态进 Flags["demonGuard"]
  if _demonGuard==1: return Steady
  if Phase==MajorConsummate && Foundation<T_CONSUM_FND(70): return Steady
  if Phase==Bottleneck && Insight>=COMP_INSIGHT_GATE(22): return Comprehend
  if arch==Martial && cult.innerDemon < DEMON_SAFE(40): return Fast
  if 当前节点无奇遇暴露 && roamCooldown到期: return Roam
  return Steady
```

滞回状态 `demonGuard` 进 Flags 随 Dict 深拷（零新 Clone 字段）。**情境化分叉**：同一 mode 在不同 Phase/innerDemon 下喂不同 A2 阈值闸（Fast 在 innerDemon∈[60,79] 赌命触 80 走火 / 在 <40 安全），产组合情境而非加按钮。

### 1.3 破单调真判据（B 批判 missing 修：新增 INV-VARIETY / INV-NO-DOMINANT）

把"破单调成功"从"选项存在"改为"NPC 自由选择下实际轨迹熵 + 无单一占优动作"：
- **INV-VARIETY**（替代 D 的 INV-TEMPO-3 锁mode测试）：自由贪心跑 N=20 不同 persona×1000tick，断言 DailyMode 序列香农熵 ≥ H_MIN 且不存在 > K_RUN 长单一 mode 连段。锁 mode 测试**保留但降级为辅助**（只证 mode 有差异，不证不塌缩）。
- **INV-NO-DOMINANT**（B 批判 missing 修，直击鬼谷"最优循环"）：1000tick 断言任意单一动作（含闭关 §2、含任意 DailyMode）长程选取频率不塌到 0、不占优 > DOM_CAP。闭关避险刷点（见 §2.6）由此 INV 兜底。

## 2. 闭关时序 — 单点 wake + 中途不Tick + 出关一次性补 Age（B 批判 P1/P10 收敛）

### 2.1 唯一模型（源码强制，作废 A3 全段）

闭关 = **延长 NextActAt 到未来绝对 tick WakeAt 的特殊 Train 子动作**，复用既有最小堆"沉底"，到 WakeAt 由 PopMin 唤醒走 SeclusionResolve。**不退出 Scheduler、不加 Scheduler 字段、不改 Lifecycle.Tick 行为**。源码论证：secluded 角色 NextActAt=WakeAt（远未来）→ 在 (At,Id) 堆中排到所有常态 At≈Clock+10 之后 → WakeAt 前永不被 PopMin → `Lifecycle.Tick`(World.cs:105) 不调用 → **Age 中途不 Tick（物理强制，非选择）**。

### 2.2 时长公式（全整数，不耗任何 RNG，作废 D主体 5×interval 量级）

```
WorkUnits      = v1.2 per-path curve: 从当前flatIndex推进到TargetSubLevel边界所需work
Duration       = SeclusionBaseTicks(60) + StepWork(8)×WorkUnits − InsightSpeedup
InsightSpeedup = clamp(Insight/2, 0, SeclusionBaseTicks/2)
WakeAt         = EnterClock + Duration
Duration       = max(1, Duration)   // 防御 clamp
```

### 2.3 折寿 AgeCost（B 批判 P5 修：乘除两端统一 interval，禁双计）

批判员证实：D集成 `AgeCost=ActionInterval(actor)×ceil(Duration/ActionIntervalBase)` 乘端含 jitter、除端用 Base 不一致，且与 A3"Age仍Tick"会双计。**修复**：
```
AgeCost = ActionInterval(actor) × ceil(Duration / ActionInterval(actor))   // 乘除同一含jitter定义
        × FoldLifeFactor / 100   // per-path: 默认100, 体修/魔修>100(闭关折寿流派)
```
**唯一计入点 = 出关 SeclusionResolve 单点**，禁止与 Lifecycle.Tick 双计（中途不 Tick 已物理保证）。寿尽判定：`Age += AgeCost` 后立即 `Age >= Lifespan + LifespanBonus → CharacterDied`（"闭关到死"是合法确定性终态，不会中途凭空猝死）。

### 2.4 闭关收益落点统一（B 批判 P6/minor 修：作废三套货币）

裁决：闭关收益 = **WorkUnits 推进 + A2 破障四法的 BreakAid**，二者对齐：
- 出关 work 达 SubLevel 边界 → 走 v1.2 渡劫机评劫（占 1 次 cultRng.Split 子流）。
- BreakAid += min(secludeStreak×SECLUDE_BREAKAID_PER_K(4), 30)，对齐 A2-FINAL §3.5 "闭关连续K→BreakAid"。
- 作废 D主体 secludeStreak 独立累加货币 / A3 secludeUntilTick 货币的三套并存。

### 2.5 闭关期 spar/奇遇（确定性可控）

- **奇遇暴露=0**（闭门碰不到外缘，RoamExposureMul=0）；但 **Epiphany/BottleneckLoosen 仍可点火**——**但见 §2.6 必须加递增成本防刷点**。
- **被 spar**：闭关者仍在 BuildContext.Nearby（位置没变，其他角色决策流不变→off-path 零扰动），SparAction 命中 secluded target **短路为 no-op + Disturb++**（不结算胜负/不改关系，加法可交换→同 tick 多发起者顺序无关）。

### 2.6 破闭关避险刷点（B 批判 P9 修：直击鬼谷"最优循环"）

批判员证实：闭关屏蔽风险奇遇却保留纯收益 Epiphany/BottleneckLoosen + BreakAid 累积 → "能闭关就闭关"占优，架空奇遇体系。**修复 = 闭关递增边际成本**：
```
secludeStreak 越高:
  FoldLifeFactor 阶梯 +5%/streak (clamp 150)   // 枯坐折寿加剧(仙侠"闭关易生心魔")
  innerDemon += SECLUDE_DEMON_PER_STREAK(3)/streak  // 枯坐生心魔
  Epiphany/BottleneckLoosen 点火权重 ×(100−streak×10)/100 (下限0)  // 闭关顿悟边际递减
```
由 **INV-NO-DOMINANT**（§1.3）兜底：断言"一直闭关"非占优（其他动作频率不塌 0）。

### 2.7 被打扰提前出关（留 G5，默认惰性记账）

默认 **惰性 B 方案**：Disturb 只累加，到 WakeAt 正常唤醒时若 `Disturb >= DisturbTolerance` 判"被打扰出关"。**禁围殴折寿套利**（B 批判 P5）：被打扰 AgeCost 须 ≥ 正常 AgeCost（骚扰不降低总折寿），否则"围殴反而少折寿"成廉价破坏策略。提前重投(方案A，需陈旧项过滤)+per-path DisturbTolerance 留 G5 增强。

### 3.1 佛修破戒修正（B 批判 P11/minor 修：NPC 自动驾驶可玩性）

## 4. 奇遇频率 cap 收敛（B 批判 P2/P7 修：下界够不到 + 护栏管错对象）

### 4.1 GlobalCap 按人口缩放（B 批判 P2 修：结构性下界冲突）

### 4.2 收益护栏扩到全突破货币（B 批判 P7 修：滚雪球刷点）

- lifespanBonus(前辈遗泽)硬 OncePerLife + LEGACY_LIFESPAN_MAX=200 单次小额；BreakAid 经 BREAK_AID_CAP 钳。
- **INV-TEMPO-2 红时收敛策略写明**：首选降 ENCOUNTER_PROGRESS_RATIO，次选降缩放后 GlobalCap，第三 per-category cap。

### 4.3 太吾双面教训（B 批判 P4 修：铺路税）

批判员证实：太吾差评不止"奇遇太少"，更是"奇遇获取过程重复"（铺路税）。**修复**：

### 4.4 内容池规模下限量化（B 批判 missing 修：24 条撑不起长跑）

批判员证实：24 条 storylet 在 [3,8]/100tick 长跑会反复撞，QBN 退化为"小池换顺序"。**修复 = 把池规模作可证伪量**：

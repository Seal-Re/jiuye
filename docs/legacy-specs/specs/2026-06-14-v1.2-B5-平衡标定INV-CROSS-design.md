## 1. 问题量化（实读为据）

战力公式（`PowerEngine.Evaluate`）：
```
power = clamp( BaseSum ×[mods] × RealmMultipliers[realmIdx] / 10 ×[postmuls], 0, cap )
其中 BaseSum = Σ_term (weight + flagStep) × Resolve(src)；Resolve: stat:* / realm / sumArtPower / res:* / derived:*
```

**失衡双源**（实证）：
1. **RealmMultipliers 未跨路对齐**：同 UT12 顶 realm 的 mul 实测 `丹5 / 法44 / 鬼55 / 毒蛊56 / 驭兽70 / 体73 / 妖88 / 音106 / 符123 / 器150 / 因果200 / 佛225 / 剑270 / 血300 / 雷340` → **5→340 = 68× 失真**（critique P1：UT8 3~72、UT10 3~180、UT12 5~340）。
2. **BaseSum 未对齐**：各路 term 权重独立调（stat 权重/sumArtPower/资源项不一）→ CLI 实证 realm0 战力 `丹8 / 阵8 / 音195 / 体146`（同 realm0=UT0 差 24×）。

**根因**：A.0 每路 RealmMultipliers + PowerFormula 是 build 期各自拟的，从无跨路基准。**同 UT ≠ 同战力** → 碾压闸（UT 差→必胜）失效、寿元/劫绑错 UT（丹 UT12 吃 UT12 寿元荒谬）。

## 2. INV-CROSS 契约（沿用 A1 §5，精化为可测）

- **C1 战斗路同 UT 平价**：任一 UT 上，任两条**战斗路**的典型角色 1v1 胜率 ∈ **[40%, 60%]**。
- **C2 碾压单调**：UT 差 ≥ 2 → 高 UT 必胜（要求 UT 真反映战斗当量，C1 保证）。
- **C3 辅助路豁免但 UT 诚实**：辅助路（战斗弱是设计）**不进 C1 平价要求**，但其 `UnifiedTierOf` 必 = 真实战斗当量 UT（即"丹修锚 UT8"意味其 1v1 ≈ UT8 战斗路下限，非 UT12）。地位走 path-local rank（丹帝/符王，与 UT 解耦）。

**胜负模型**（标定期对拍用）：`SparAction.EffectivePower` = per-path power × 软情境 adj（clamp ±P0/4）。同 UT 双方 power 越近 → 胜率越近 50%。故 **C1 的代理指标 = 同 UT 典型 power 落在同一带（±X%）**；最终以对拍胜率验收。

## 3. 典型角色定义（标定 fixture，确定性）

标定/对拍用"典型角色 @ (path p, UT u)"：
- **四维**：中庸 Σ=80（如 Force/Internal/Constitution/Insight 各 20；或按生成期分布中位）。统一所有路同四维 → 隔离 path 变量。
- **realm**：UT u 在 p 的代表 flatIndex = 该 UT 段**段首**（`RealmProjection.Encode(major(u),0)`）。
- **功法**：标准 loadout = 每类目取 PickMin 个 tier 中位 art（统一策略，sumArtPower 可比）。
- **资源**：res:* 取 Initial（生成期初值）。**flags 空**（无情境台阶）。
- **道心**：不进战力（R3），不涉。

> 典型角色是**比较基准**，非实战分布。实战分布另由生成期 Σ=80 随机 + 奇遇/功法分化，标定只保证"中庸基线"平价。

## 4. 标定 harness（三件，read-only 诊断 + gate）

1. **PowerMatrix dump**：遍历 21 路 × 各路 UnifiedTierOf 去重 UT → 构造典型角色 → `PowerEngine.Evaluate` → 输出 `(path, UT, BaseSum, mul, power)` 矩阵。**揭示真实 spread**（设计基于真数据非估计）。
2. **Duel sim**：对每个有 ≥2 战斗路的 UT，两两典型角色跑 N 次 `SparAction`（含软情境随机）→ 统计胜率。
3. **Convergence report**：列出违反 C1（胜率∉[40,60]%）的 (UT, pathA, pathB) + 偏离倍率；驱动迭代。

harness 落 `tests/` 诊断 + 后续转 gate 断言（INV-CROSS test）。

## 5. 归一化法（核心：解析校准 RealmMultipliers）

**目标曲线 target(UT)**：选一条 canonical 战力当量曲线（决策 D1，见 §8）。推荐 = **剑修（范式路）的典型 power(UT)** 作基准（范式路已是参考实现，曲线平滑 1.4-1.5×/阶）。

**解析校准**（战斗路）：典型 `power_p(realm i) = BaseSum_p(i) × mul_p[i] / 10`。令其 == target(UT_i)：
```
mul_p[i] := round( target(UT_i) × 10 / BaseSum_p(i) )
```
- BaseSum_p(i) 由 harness 实测（典型角色 @ realm i 的段 1 求值，注意含 src="realm"=RealmIndex 随阶变 → 逐 realm 算）。
- 校准后**校验严格递增**（target 随 UT 增 → mul 随之增；若某阶因 BaseSum 跳变破递增 → 微调 target 或 BaseSum 项权重，记录）。
- **保留各路"手感"**：只动 mul（战力级别），不动 term 结构（路的战斗"风格"=哪些 src 进 BaseSum 不变）。若 BaseSum 本身畸高/低致 mul 失真（如丹），归类辅助路（§6）。

**迭代**：dump→校准 mul→duel 验胜率→未过则微调 target 带宽/个别路 → 收敛。

## 6. 辅助路战斗当量重锚（解 A1.4）

- **辅助路判定**（决策 D3）：战斗 power 显著低于同 UT 战斗路下限（典型：丹/符/器/阵 — 战力来自资源/地位非直接战斗）。
- **重锚**：测辅助路典型 power → 在 target(UT) 上反查其落点 UT* → 设其 `UnifiedTierOf` 顶 = UT*（如丹典型 power ≈ target(UT8) → 丹战斗当量顶 UT8，非 UT12）。**只动 UnifiedTierOf**（auditor M1：不动 RealmMultipliers 保递增；境界稿 §11.4）。
- **地位保留**：丹帝/符王 realm **名**不变（A1.3 已定），走 path-local rank（itemTier/recipeCount…），与 UT 解耦。
- **连锁**：UT 降→寿元/劫档随之合理（丹吃 UT8 寿元非 UT12）。寿元/劫数值属阶段4，本稿只定 UT 锚。

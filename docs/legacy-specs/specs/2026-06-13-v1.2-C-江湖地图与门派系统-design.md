## 2. 程序化江湖地图（`Jianghu.Geo`）

### 2.1 三层模型（替换 3 抽象节点，NodeId 复用不重编号）
```
Realm（江湖根）
 └─ Region[]（区域 6~12：中原/塞外/江南/西域/苗疆…，门派据此占地利）
     ├─ Site[]（地点 3~8/区：城镇/客栈/关隘/渡口/门派山门/秘境/资源点，Site.Id = NodeId，角色实际所在）
     └─ 地利三维整数：Wealth/QiDensity/Strategic [0,100]
```
- `Site.Id` **复用 `NodeId` 同主键空间**（不重编号）→ v1.0 `Nodes/NodeCount/AtNode/Move` 全签名兼容。NodeId 连续 0..N-1（前缀和按区域分配，`RegionOf`=整数二分），保 `TravelAction.cs:17` 的 `0<=To<NodeCount` 边界检查不破。
- 地理/资源/归属全走**侧表**（键 NodeId）：`NodeGeo`（GeoRec record 值拷）、`AdjacencyGraph`、`ResourceNode`、`SiteOwnership`。WorldNode record 一字不改。
- 秘境/洞天 = `SiteKind.Secret` 隐藏 Site，显形态走 `World.RevealedSecrets:HashSet<int>`（小集合深拷），**地图拓扑本体生成后冻结不可变**（`WorldMap.Clone()=>this` 浅拷安全，对齐 `World.cs:159` Nodes 浅拷纪律）。

### 2.2 生成算法（mapRng=root.Split(7)，纯整数，随机有上限）
```
输入 GeoConfig{ GridW,H, RegionMin/Max, SiteMin/MaxPerRegion, MaxDegree, MaxSecrets, RedundancyPct, ConnectRadius, SecretQiThreshold }
1 区域数 = mapRng.NextInclusive(RegionMin,RegionMax)；区域中心=整数 grid-jitter（cellOrigin + mapRng.NextInt(cellW/2)），曼哈顿/切比雪夫整数距离，无浮点
2 区域类型 VariedSelector 加权抽（Rarity 整数，不重复优先）；地利三维 = NextInclusive 经地形整数偏置表后 clamp[0,100]
3 区内 Site 数 = NextInclusive(SiteMin,SiteMax) clamp≤MaxSitesPerRegion；SiteType 从区域 SiteComposition 加权抽；NodeId 全局单调递增分配
4 秘境：QiDensity≥SecretQiThreshold 且 mapRng.NextInt(100)<阈值 → 建 Secret Site；全图秘境 ≤ MaxSecrets（硬上限优先于随机）
5 资源点：每 Site 按 ResourceTable 加权抽 0..K 个 ResourceNode；Grade=NextInclusive(GradeMin,GradeMax)
6 拓扑=确定性 Kruskal MST + 受控冗余：候选边=曼哈顿距离≤ConnectRadius 的 Site 对，按 (整数权 asc, fromId asc, toId asc) 三级排序；并查集建 MST（保连通无孤岛）；同序补冗余边至 |E|≥MST+MST*RedundancyPct/100 且每 Site degree≤MaxDegree；每相邻区域至少 1 跨区桥；BFS 连通断言 fail-fast
7 邻接表每项升序 Sort（Dictionary 仅索引，裁决前必排序，对齐 World.AtNode World.cs:74）
```
**不变量**：INV-MAP-CONNECTED（并查集全连通）/ 区域连通 / degree∈[1,MaxDegree] / NodeId 唯一连续 / mapRng 是 Split(7) 私有流，决策/其他系统绝不碰，map-off 不构造不消费。

### 2.3 节点语义喂决策（修复"死数据"批判，C.0 最薄闭环）
> 批判员指出：节点扩到 N 个有 type/资源/DangerTier，但角色唯一动作仍 Travel 且 `to=node+1`，秘境/资源全是装饰数据。修复二选一，**C.0 采纳 (a)+(b) 最薄版**：
- **(a) RuleBrain.Travel 目标导向选邻居**（已是受控改动，见 §0）：`Reachable[0]` 在 map-on 时按"邻居资源/Hub 权重"排序后取首（仅加权排序，非寻路）。map-off `Reachable=[Node+1]` 等价旧。
- **(b) 最薄 `HarvestAction`**：角色在有 `ResourceNode` 的 Site 可采集，产 `ResourceHarvested` 事件。**A 落地时**经 `CultivationState.ApplyResource` chokepoint 喂修炼；**A 未落地时降级**：仅写 Chronicle flavor + Treasury（门派侧整数），不碰四维。
- **秘境进入**：`EnterSecret` 经整数 EntryGate 门控，产 `SecretRevealed`。A 未落地时仅显形+Chronicle，不发奇遇资源。

## 3. 门派系统（`Jianghu.Faction`，侧表聚合）

### 3.1 数据驱动定义（加新门派/关系=纯数据，修复"硬编码反例"批判）

### 3.2 运行态聚合（独立 SectLedger，不进 Sect 单例）

         RegionId? HomeRegion, List<SiteId> ControlledSites, int Reputation, int Treasury, bool Active }

            Dictionary<int,int> _bySite/_byMember(索引), Dictionary<(int,int),int> _relations(门派间有向[-100,100]) }

- **层级整数 rank**：掌门5/长老4/真传3/内门2/外门1/记名0，存 `SectMembership.Rank`。INV-SECT-1：每门恰 1 掌门。

- **确定性铁律**：每个新集合显式指定裁决遍历前规范排序键（Faction 按 Id、关系边按 (from,to)、资源按 SiteId），键在 Clone 续跑下稳定（用整数全序，不用插入顺序）。

### 3.3 拜师/晋升/叛门（角色可驱动反馈环，修复"堆数据无深度"批判）

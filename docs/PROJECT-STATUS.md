## 5. 优先级建议（"现在优先设计/做什么"）

**先收尾在建的，再补最大缺口，别再开新摊。**
1. **收尾 A.1 境界竖切**(A1.3 命名 / A1.5 三轴查询 / A1.6 gate + **A1.4 决断**：补做或显式 defer+标依赖)。别留半成分支烂。
2. **补最大功能缺口二选一**：
   - **(a) 平衡标定 INV-CROSS** — 没它涌现战斗失衡，21 路名存实亡。设计标定工具(全路×UT 战力矩阵 dump + 对拍 harness)+ 迭代收敛。**最影响"游戏成立"**。

3. **系统集成层设计** — 修炼+戏剧+地图+门派如何在一个 World.Advance 里协同跑出"一局江湖"。各系统设计齐了，组合没设计。
4. **可视化按用户分轨**：游戏世界(tile/角色/物品)=像素(Pillow)；**UI/界面=精细化古风(非像素，SVG/HTML-CSS 水墨/卷轴，贴合背景)**——见 pixel 规则待补 UI 轨。viewer 接 sim JSON 导出。
